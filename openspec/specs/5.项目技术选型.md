# 5. 项目技术选型

> 基于 1.spec.md 功能需求、2.design.md 架构设计、6.模块职责细化及技术细节的约束条件

---

## 一、核心技术栈

### 1.1 编程语言

**选型: Python 3.11+**

**理由**:
1. **生态丰富**: 大模型 API SDK(OpenAI/Anthropic/Qwen 等)均提供官方 Python SDK
2. **异步支持**: `asyncio` 原生支持单线程异步并发,满足 SDPJDetector 的 I/O 密集型并发需求
3. **性能提升**: Python 3.11 比 3.10 快 25%(Faster CPython 项目),比 3.9 快 40%
4. **TaskGroup 支持**: `asyncio.TaskGroup`(3.11 新增)提供更好的异常传播与任务取消机制,简化 StateScheduler 并发调度逻辑
5. **快速开发**: 动态类型、丰富的第三方库,适合毕业设计快速迭代
6. **跨平台**: 支持 Windows/Linux/macOS,满足"一键部署"需求

**版本说明**:
- **最低要求**: Python 3.11(必需,因 StateScheduler 使用 asyncio.TaskGroup)
- **推荐版本**: Python 3.11.8+(最新稳定版)

## 二、分层技术选型

### 2.1 用户界面层(UI Layer)

#### 2.1.1 CLI 模块

**选型: Click 8.1+**

**理由**:
- 声明式命令定义,代码简洁
- 自动生成帮助文档
- 支持参数校验与类型转换
- 轻量级(无额外依赖)

**备选**: argparse(Python 标准库,但 API 冗长)

---

#### 2.1.2 WebUI 模块

**前端选型: Vue 3.4+ + Element Plus 2.8+ + ECharts 5.5+**

**理由**:
- Vue 3 组合式 API 适合中小型项目
- Element Plus 提供开箱即用的图表组件(ECharts 集成)
- 轻量级,打包后体积小

**状态管理**: Pinia 2.1+(管理用户登录态、检测任务进度等全局状态)

**HTTP 客户端**: axios 1.7+

**理由**:
- Vue 生态事实标准
- 支持请求/响应拦截器(自动附加 JWT token)
- 支持请求取消、超时控制
- TypeScript 支持完善

**后端选型: FastAPI 0.115+**

**理由**:
- 原生支持异步(async/await),与 SDPJDetector 异步架构一致
- 自动生成 OpenAPI 文档
- 内置数据校验(Pydantic)
- 性能优于 Flask(基于 Starlette + Uvicorn)

**C-S 通信**:
- 协议: HTTP/1.1 + WebSocket(实时进度推送)
- 加密: TLS 1.3(生产环境,HTTPS) / HTTP(开发环境)
- 说明: 账号密码通过 TLS 1.3 加密传输,服务端接收后由 bcrypt 哈希存储;私有配置文件通过 TLS 1.3 加密传输,服务端由 SecureCommManager 加密存储
- 部署说明: 生产环境的 TLS 终止由外部反向代理(Nginx/Caddy)处理,docker-compose 配置仅用于开发环境(HTTP)

**认证方案: JWT (JSON Web Token)**
- WebUI: 用户登录成功后生成 JWT token,前端存储在 localStorage,后续请求在 HTTP Header 中携带 `Authorization: Bearer <token>`
- CLI: token 存储在本地文件 `~/.sdpj/token`(权限 600),后续命令自动读取
- 技术选型: PyJWT 2.8+

**CORS 配置**(前后端分离必选):
- 实现: FastAPI 内置 `CORSMiddleware`(无需额外依赖)
- 配置: 允许前端源(开发环境 `http://localhost:3000`,生产环境根据实际域名配置)
- 示例:
```python
from fastapi.middleware.cors import CORSMiddleware

app.add_middleware(
    CORSMiddleware,
    allow_origins=["http://localhost:3000"],  # 前端地址
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```
- Token 过期时间: 24 小时(可配置)
- 刷新机制: Refresh Token(7 天有效)

**WebSocket 实时进度推送**:
- 实现: FastAPI 内置 WebSocket 支持(starlette,无需额外依赖)
- 用途: 检测任务进度实时推送到前端

**技术细节**:
```python
# FastAPI 端点
@app.websocket("/ws/task/{task_id}")
async def websocket_endpoint(websocket: WebSocket, task_id: str):
    await websocket.accept()
    # 轮询数据库,推送任务进度
    while True:
        task = await get_task_by_id(task_id)
        await websocket.send_json({
            "status": task.status,
            "progress": task.progress,
            "message": task.message
        })
        if task.status in ["completed", "failed"]:
            break
        await asyncio.sleep(1)  # 每秒推送一次
```

**前端实现**:
```javascript
// Vue 3 组件
const ws = new WebSocket(`ws://localhost:8000/ws/task/${taskId}`)
ws.onmessage = (event) => {
  const progress = JSON.parse(event.data)
  updateProgress(progress)
}
```

**重连策略**: 前端检测断线后自动重连(最多 3 次,间隔 5 秒)

**备选**: Flask(同步框架,不适合异步场景)

### 2.2 应用控制层(Controller Layer)

#### 2.2.1 StateScheduler 模块

**状态机实现: python-statemachine 2.1+**

**理由**:
- 声明式状态定义与转移规则
- 支持状态进入/退出钩子
- 可视化状态图导出

**并发调度: asyncio.TaskGroup (Python 3.11+)**

**理由**:
- 原生异步任务组管理
- 自动异常传播与取消(任一任务失败自动取消其他任务)
- 比 asyncio.gather 更安全的错误处理

### 2.3 执行逻辑层(Execution Logic Layer)

#### 2.3.1 SDPJDetector 模块

**异步并发: asyncio + aiohttp 3.9+**

**理由**:
- `asyncio`: Python 原生异步运行时,单线程事件循环
- `aiohttp`: 异步 HTTP 客户端,支持连接池与超时控制
- 满足"单线程异步并发"约束(SDPJDetector.md L65)

**aiohttp 配置**:
- 连接池大小: 100(可通过配置文件调整)
- 单次请求超时: 60s(适应大模型响应时间)
- 连接超时: 10s
- 总超时: 120s(含重试)

**LLM API 调用**:
- 通过 LLMAdapterLib 统一封装
- 每个适配器实现 LLMServiceInterface 定义的异步接口: `async def call(system_prompt: str, user_message: str, images=None, audios=None, videos=None, **kwargs) -> str`

---

#### 2.3.2 TaskQueueManager 模块

**选型: asyncio.Queue**

**理由**:
- Python 原生异步队列
- 支持 FIFO
- 线程安全(虽然本系统单线程,但保留扩展性)

**配置**:
- maxsize: 1000(防止突发大量任务导致内存无限增长)
- 背压策略: 队列满时阻塞新任务提交,返回 HTTP 429(Too Many Requests)

**备选**: Redis Queue(过度设计,增加部署复杂度)

---

#### 2.3.3 EventLogger 模块

**选型: Python logging 标准库 + structlog 24.1+ + 异步日志处理**

**理由**:
- `logging`: 标准库,零依赖
- `structlog`: 结构化日志(JSON 格式),便于后续日志分析
- 支持多输出目标(控制台/文件/内存缓冲)

**异步日志方案**(避免阻塞 SDPJDetector 事件循环):
- **推荐**: `logging.handlers.QueueHandler` + 后台线程刷盘
  - 使用 `contextvars` 传递请求上下文(user_id/task_id)
  - 内存缓冲 + 批量写入,减少 I/O 频率
- **备选**: `aiofiles` 实现异步文件写入(增加依赖,性能提升有限)

**日志级别**: DEBUG / INFO / WARN / ERROR

---

#### 2.3.4 异常处理模块

**选型: Python 标准库 + FastAPI 异常处理中间件**

**全局异常捕获**:
- FastAPI: 使用 `@app.exception_handler` 捕获全局异常
- CLI: 使用 `try-except` + 状态机异常转移

**异常分类**:
```python
class SDPJException(Exception):
    """系统基础异常"""
    pass

class NetworkError(SDPJException):
    """网络错误(LLM API 调用失败)"""
    pass

class BusinessError(SDPJException):
    """业务错误(数据校验失败、权限不足)"""
    pass

class SystemError(SDPJException):
    """系统错误(数据库连接失败、文件读写失败)"""
    pass
```

**异常处理策略**:
- NetworkError: 重试 3 次(由 LLMInterface 的 tenacity 处理)
- BusinessError: 返回错误信息给用户,不重试
- SystemError: 记录日志,状态机转移到 ERROR 状态

**数据库事务回滚**:
- 使用 SQLAlchemy 的 `async with session.begin()` 自动回滚

#### 2.3.5 ReportManager 模块

**可视化数据准备: 无额外库(纯 Python 数据结构)**

**理由**:

#### 2.3.5 SecureCommManager 模块

**选型: cryptography 42.0+**

**理由**:
- Python 官方推荐的加密库
- 支持 AES-256-GCM(对称加密)
- 支持密钥派生(PBKDF2/Scrypt)
- 提供主流平台预编译 wheel,跨平台兼容性好

**加密方案**:
- 私有配置文件(用户 JSON 适配器配置,包含 API Key): AES-256-GCM + 随机 IV
- 说明: 账号密码由 UserCenter 的 bcrypt 哈希存储,不使用 AES 加密
- 密钥管理: 
  - 开发环境: 环境变量 `SDPJ_SECRET_KEY`(32 字节 base64)
  - 生产环境: 建议使用密钥管理服务(AWS Secrets Manager / HashiCorp Vault)或文件权限受限的本地密钥文件(`~/.sdpj/secret.key`,权限 600)
  - 密钥轮换: 支持多密钥并存,旧密钥保留 30 天用于解密历史数据,新数据使用新密钥加密

**备选**: PyCryptodome(API 复杂)

#### 2.4.2 LLMInterface 模块

**接口契约: LLMServiceInterface**

SDPJDetector 通过此接口调用 LLMInterface,封装了适配器选择和重试策略:

```python
from typing import Protocol, Optional, List

class LLMServiceInterface(Protocol):
    """SDPJDetector 通过此接口调用 LLMInterface。封装了适配器选择和重试策略。"""

    async def call(
        self,
        system_prompt: str,
        user_message: str,
        images: Optional[List[str]] = None,
        audios: Optional[List[str]] = None,
        videos: Optional[List[str]] = None,
        **kwargs
    ) -> str:
        """
        发起单次大模型调用(含 tenacity 重试逻辑,标准化错误处理)。

        参数:
          system_prompt: 系统提示词
          user_message: 用户消息
          images: List[str],每项可为 HTTP URL 或 base64 Data URI
          audios: List[str],每项可为 HTTP URL 或 base64 Data URI  
          videos: List[str],每项可为 HTTP URL 或 base64 Data URI
          **kwargs: 额外参数(如 temperature, max_tokens)
        
        返回: 模型原始响应文本
        
        异常: LLMServiceError(标准化错误,含 ErrorType)
        """
        ...
```

**重试策略: tenacity 8.2+**

**理由**:
- 声明式重试装饰器
- 支持指数退避(exponential backoff)
- 支持异步函数

**示例**:
```python
@retry(
    stop=stop_after_attempt(3),
    wait=wait_exponential(multiplier=1, min=2, max=10),
    retry=retry_if_exception_type(NetworkError)
)
async def call_llm(prompt: str) -> str:
    ...
```

### 2.5 基础设施层(Infrastructure Layer)

#### 2.5.1 数据库模块(SampleDB / ResultDB / UserDB)

**选型: SQLite 3.40+ (开发/单机部署) / PostgreSQL 15+ (生产/多用户)**

**理由**:
- **SQLite**:
  - 零配置,单文件数据库(`sdpj.db`,合并 SampleDB/ResultDB/UserDB)
  - 满足"轻量级、一键部署"需求
  - 适合毕业设计演示
  - **并发写入限制**: SQLite 使用单写者锁,多并发写入会触发 SQLITE_BUSY 错误。建议开发环境并发度 ≤3,生产环境升级 PostgreSQL
- **PostgreSQL**(可选升级路径):
  - 支持并发写入
  - 完整的 ACID 保证
  - 适合生产环境

**ORM: SQLAlchemy 2.0+ (异步模式)**

**理由**:
- 支持异步(async/await)
- 数据库无关(SQLite/PostgreSQL 切换无需改代码)
- 声明式模型定义
- 连接池 + 重试机制缓解 SQLite 写锁竞争

**异步模式避坑指南**:
1. **禁用 Lazy Loading**: 使用 `selectinload()` 或 `joinedload()` 预加载关联对象
   ```python
   # ❌ 错误:Lazy loading 在异步中会报 MissingGreenlet 错误
   user = await session.get(User, user_id)
   reports = user.reports  # 错误!
   
   # ✅ 正确:显式预加载
   stmt = select(User).options(selectinload(User.reports)).where(User.id == user_id)
   user = (await session.execute(stmt)).scalar_one()
   ```

2. **使用 NullPool**: SQLite 异步模式下避免连接池问题
   ```python
   engine = create_async_engine(
       "sqlite+aiosqlite:///sdpj.db",
       poolclass=NullPool  # 禁用连接池
   )
   ```

3. **启用 WAL 模式**: 提升 SQLite 并发写入能力
   ```python
   async with engine.begin() as conn:
       await conn.execute(text("PRAGMA journal_mode=WAL"))
   ```

4. **事务管理**: 使用 `async with session.begin()` 自动回滚
   ```python
   async with async_session() as session:
       async with session.begin():
           session.add(task)
   # 异常时自动回滚
   ```

**数据库 Schema**:
- 单文件模式: 合并为 `sdpj.db`,包含 11 张表(datasets/samples/task_groups/tasks/reports/result_data/users/resources/acl/private_configs/sessions)
- 多文件模式(可选): SampleDB/ResultDB/UserDB 分离为 3 个 `.db` 文件

**跨表外键约束**:
- SQLite 单文件模式下,外键约束由数据库强制执行
- 若使用多文件模式,以下跨库外键需在应用层校验(由 DataProcessor 负责):
  - `ResultDB.task_groups.user_id → UserDB.users.user_id`
  - `ResultDB.tasks.dataset_id → SampleDB.datasets.dataset_id`

#### 2.5.2 LLMAdapterLib 模块

**适配器接口**:
```python
class LLMAdapter(ABC):
    @abstractmethod
    async def call(
        self, 
        system_prompt: str, 
        user_message: str,
        images: Optional[List[str]] = None,      # 图片 base64 或 URL
        audios: Optional[List[str]] = None,      # 音频 base64 或 URL
        videos: Optional[List[str]] = None,      # 视频 base64 或 URL
        **kwargs
    ) -> str:
        """统一的异步调用接口,支持多模态输入"""
        pass

    @abstractmethod
    def get_error_type(self, exception: Exception) -> ErrorType:
        """统一错误分类"""
        pass
```

**通用适配器引擎**:
- 无内置适配器,所有 LLM(包括 OpenAI/Anthropic/通义千问等)均通过用户 JSON 配置接入
- 支持的 request_format: `openai`(OpenAI API 格式), `anthropic`(Anthropic API 格式), `custom`(自定义模板)
- 根据 request_format 自动适配不同 LLM 的 API 格式

**依赖库**:
- aiohttp 3.10.5(异步 HTTP 客户端,调用 LLM API)
- 无需安装各 LLM 的官方 SDK(openai/anthropic/dashscope 等)

**用户适配器配置**:
- 格式: JSON 配置文件(不支持代码执行,确保安全)
- 存储: 通过环境变量 `SDPJ_ADAPTERS_DIR` 指定目录(默认 `~/.sdpj/adapters/`)
- 安装: `sdpj adapter install <file.json>` 或手动复制到目录

**JSON 配置示例**(OpenAI GPT-4):
```json
{
  "name": "openai-gpt4",
  "api_url": "https://api.openai.com/v1/chat/completions",
  "api_key": "sk-xxx",
  "request_format": "openai",
  "model": "gpt-4",
  "headers": {
    "Content-Type": "application/json"
  },
  "timeout": 60
}
```

**JSON 配置示例**(Anthropic Claude):
```json
{
  "name": "anthropic-claude",
  "api_url": "https://api.anthropic.com/v1/messages",
  "api_key": "sk-ant-xxx",
  "request_format": "anthropic",
  "model": "claude-3-opus-20240229",
  "timeout": 60

```

**支持的 request_format**: `openai`(OpenAI API 格式), `anthropic`(Anthropic API 格式), `custom`(自定义模板)

**多模态支持**:

- `supported_types`: 支持的模态类型,可选 `text`/`image`/`audio`/`video`
- `image_format`: 图像传输格式,可选 `base64`(编码) / `url`(URL 链接)
- `image_detail`: OpenAI/DeepSeek 的图片处理精度,可选 `low`(85 tokens) / `high`(85+170/tile) / `auto`(默认,自动选择)
- `audio_format`: 音频传输格式,可选 `base64` / `url`
- `video_format`: 视频传输格式,可选 `base64` / `url`
- `max_file_size_mb`: 单个文件最大体积限制(MB)
- `max_images_per_request`: 单请求最大图片数量(OpenAI=10, DeepSeek≈10, Anthropic=100)
- `supported_mime_types`: 支持的 MIME 类型,用于前置校验(OpenAI 不支持 HEIC,Gemini 不支持部分 webp)
- `preprocessing.resize_strategy`: 图片缩放策略,可选 `fit`(等比缩放) / `pad`(填充) / `crop`(裁剪) / `none`(不缩放)
- `preprocessing.max_resolution`: 最大分辨率限制,直接影响 Token 消耗(Anthropic 推荐 ≤1568px,OpenAI 推荐 ≤2048px)
- 默认仅支持文本模式,用户需手动配置 `multimodal` 字段启用多模态
