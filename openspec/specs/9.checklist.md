# 9. 任务检查清单与开发规范

> 本文档定义所有开发任务的验收标准、OpenSpec合规性检查规则、任务粒度原则

---

## 一、任务粒度定义原则

### 1.1 2小时原则（Feature-Task-Planning）

**核心规则**: 单个任务的实际编码时间不超过 2 小时

**任务拆分标准**:
- ✅ **合格任务**: "实现 UserDB.创建用户方法 + 单元测试"（预计 1.5h）
- ❌ **过大任务**: "完成整个 UserDB 模块"（预计 8h）
- ❌ **过小任务**: "导入 SQLAlchemy"（预计 5min）

**拆分维度**:
```
模块开发 = 接口定义(0.5h) + 核心逻辑实现(1h) + 单元测试(0.5h)
```

### 1.2 模块内任务拆分模板

每个模块按以下粒度拆分：

**阶段1: 接口与模型定义**（约0.5-1h）
- 定义 Protocol/ABC 接口
- 定义 SQLAlchemy ORM 模型（数据库模块）
- 定义数据类/类型注解

**阶段2: 核心逻辑实现**（按职责拆分，每个职责≤2h）
- 每个模块的"具体职责"章节中的每条职责作为一个任务
- 示例: UserCenter 有 19 条职责 → 拆分为 19 个任务

**阶段3: 单元测试**（与实现任务1:1对应）
- 每个实现任务对应一个测试任务
- 测试覆盖率要求: ≥80%

## 二、OpenSpec 规范合规性检查

### 2.1 强制检查项（每个任务完成后必查）

#### ✅ Checkpoint 1: 规格文档验证
```
□ 实现前已阅读对应模块的职责细化文档
□ 实现的功能在"具体职责"章节中有明确定义
□ 未实现"不需要的"边界中列出的功能
□ 调用的下层接口在"依赖模块"中有声明
```

#### ✅ Checkpoint 2: 接口契约验证
```
□ 实现了"应实现接口"中声明的所有接口
□ 仅调用了"调用接口"中声明的下层接口
□ 未越层调用（如执行逻辑层直接调用基础设施层）
□ 接口签名与依赖关系图.puml 中的定义一致
```

#### ✅ Checkpoint 3: 边界声明验证
```
□ "不负责的边界"中的功能确实未实现
□ 职责范围内的功能已完整实现
□ 错误情形按规格定义处理
□ 后置条件得到满足
```

#### ✅ Checkpoint 4: 技术细节验证
```
□ 使用的技术栈符合 5.项目技术选型.md
□ 目录结构符合 8.项目目录结构设计.md
□ Python 版本 ≥ 3.11（必需）
□ 异步模块使用 asyncio（如 SDPJDetector）
```

### 2.2 规格一致性测试

**测试驱动验证**:
```python
# 示例: 验证 UserCenter 的接口契约
def test_user_center_interface_compliance():
    """验证 UserCenter 实现了 UserCenterInterface"""
    assert isinstance(user_center, UserCenterInterface)
    
def test_user_center_dependencies():
    """验证 UserCenter 仅调用允许的下层接口"""

```

## 四、测试标准

### 4.0 测试总体策略

**测试时机原则**:
- **单元测试**: 每个模块实现过程中同步进行，每个模块完成后必须通过单元测试
- **集成测试**: 阶段三所有功能完工后进行，验证各层模块间的协作
- **端到端测试**: 阶段三所有功能完工后进行，验证完整业务流程

> ⚠️ **重要提醒**: 集成测试和端到端测试不是边开发边进行的，必须等到所有模块功能开发完成后再执行。在此之前，各模块的质量保证依靠单元测试。

**测试优先级**:
```
单元测试（最优先）> 集成测试 > 端到端测试
```

**各模块测试要求**:
```
□ 每个模块必须有单元测试（覆盖率 ≥ 70%）
□ 核心模块（SDPJDetector, StateScheduler, DataProcessor）覆盖率 ≥ 80%
□ 集成测试在所有模块完成后统一执行
□ 端到端测试在所有模块完成后统一执行
```

## 六、规格变更流程

### 6.1 何时需要变更规格

**触发条件**:
```
□ 实现过程中发现规格定义不清晰
□ 发现规格中的接口契约不合理
□ 发现规格遗漏了必要的职责
□ 技术选型需要调整
```

### 6.2 规格变更步骤

1. **停止实现** - 不要基于错误的规格继续编码
2. **记录问题** - 使用Git commit记录规格问题
   ```bash
   git add <修改的规格文档>
   git commit -m "spec: 修正UserCenter职责6返回值定义
   
   问题: 职责6'登录凭据校验'未明确返回用户ID
   修正: 输出改为'校验结果(通过时附带用户ID)'

   Refs: SDPJ-System/openspec/specs/6.模块职责细化及技术细节/4.Abstract Driver Layer/UserCenter.md"
   ```
3. **提出修正** - 更新规格文档
4. **验证影响** - 检查变更是否影响其他模块
5. **同步实现** - 基于新规格重新实现

## 七、常见问题检查清单

### 7.1 接口契约违规

❌ **错误示例**: 执行逻辑层直接调用基础设施层
```python
# ❌ SDPJDetector 直接调用 SampleDB（越层）
class SDPJDetector:
    def __init__(self, sample_db: SampleDB):
        self.sample_db = sample_db  # 违规！
```

✅ **正确示例**: 通过抽象驱动层调用
```python
# ✅ SDPJDetector 通过 DataProcessor 调用
class SDPJDetector:
    def __init__(self, data_processor: DataProcessorInterface):
        self.data_processor = data_processor  # 正确
```

### 7.2 边界声明违规

❌ **错误示例**: 实现了"不需要的"功能
```python
# ❌ UserCenter 维护登录会话（边界声明：不维护）
class UserCenter:
    def __init__(self):
        self.sessions = {}  # 违规！
```

✅ **正确示例**: 遵守边界声明
```python
# ✅ 登录会话由 AccountManager 维护
class AccountManager:
    def __init__(self):
        self.sessions = {}  # 正确
```

### 7.3 异步模式错误

❌ **错误示例**: SQLAlchemy 异步模式下使用 Lazy Loading
```python
# ❌ 会报 MissingGreenlet 错误
user = await session.get(User, user_id)
reports = user.reports  # 违规！
```

✅ **正确示例**: 显式预加载
```python
# ✅ 使用 selectinload 预加载
stmt = select(User).options(selectinload(User.reports)).where(User.id == user_id)
user = (await session.execute(stmt)).scalar_one()
```

### 7.4 Python异步开发常见错误

❌ **错误4: 忘记await异步函数**
```python
# ❌ 错误：返回coroutine对象，未执行
result = async_function()

# ✅ 正确
result = await async_function()
```

❌ **错误5: 在异步函数中使用同步阻塞调用**
```python
# ❌ 错误：time.sleep阻塞事件循环
async def process():
    time.sleep(5)

# ✅ 正确
async def process():
    await asyncio.sleep(5)
```

❌ **错误6: 未正确处理asyncio.TaskGroup异常**
```python
# ❌ 错误：未捕获ExceptionGroup
async with asyncio.TaskGroup() as tg:
    tg.create_task(task1())
    tg.create_task(task2())

# ✅ 正确：使用Python 3.11+的except*语法
try:
    async with asyncio.TaskGroup() as tg:
        tg.create_task(task1())
        tg.create_task(task2())
except* Exception as eg:
    for exc in eg.exceptions:
        logger.error(f"Task failed: {exc}")
```

❌ **错误7: 异步生成器未正确关闭**
```python
# ❌ 错误：消费者提前退出时finally不会执行
async def stream_data():
    conn = await db.connect()
    try:
        async for row in conn.execute(query):
            yield row
    finally:
        await conn.close()

# ✅ 正确：使用async with
async def stream_data():
    async with db.connect() as conn:
        async for row in conn.execute(query):
            yield row
```

### 7.5 SQLAlchemy异步模式常见错误

❌ **错误8: session关闭后访问ORM对象属性**
```python
# ❌ 错误：DetachedInstanceError
async with async_session() as session:
    user = await session.get(User, user_id)
reports = user.reports  # 错误！

# ✅ 正确：在session内预加载
async with async_session() as session:
    stmt = select(User).options(selectinload(User.reports))
    user = (await session.execute(stmt)).scalar_one()
    reports = user.reports  # 正确
```

❌ **错误9: SQLite异步未启用WAL模式**
```python
# ❌ 错误：并发写入性能差
engine = create_async_engine("sqlite+aiosqlite:///sdpj.db")

# ✅ 正确：启用WAL模式
engine = create_async_engine("sqlite+aiosqlite:///sdpj.db")
async with engine.begin() as conn:
    await conn.execute(text("PRAGMA journal_mode=WAL"))
```

### 7.6 五层架构常见错误

❌ **错误10: 抽象驱动层直接返回ORM对象**
```python
# ❌ 错误：DataProcessor返回SQLAlchemy ORM对象
class DataProcessor:
    async def get_dataset(self, dataset_id: str) -> Dataset:
        return await self.sample_db.get_dataset(dataset_id)

# ✅ 正确：返回DTO
class DataProcessor:
    async def get_dataset(self, dataset_id: str) -> DatasetDTO:
        dataset = await self.sample_db.get_dataset(dataset_id)
        return DatasetDTO.from_orm(dataset)
```

❌ **错误11: 执行逻辑层依赖具体实现而非接口**
```python
# ❌ 错误：依赖具体类
class SDPJDetector:
    def __init__(self, data_processor: DataProcessor):
        self.data_processor = data_processor

# ✅ 正确：依赖Protocol接口
class SDPJDetector:
    def __init__(self, data_processor: DataProcessorInterface):
        self.data_processor = data_processor
```
