### 2.2 应用控制层 (control/)

```
control/
├── __init__.py
├── state_scheduler_interface.py    # StateScheduler 接口
└── state_scheduler.py              # 系统状态管理及调度控制
    # 包含:
    # - SystemStateMachine (python-statemachine)
    # - TaskScheduler (asyncio.TaskGroup)
```

**职责说明**:
- **应用控制层**: 负责全局状态机管理和任务调度编排,协调各执行逻辑层模块的工作流
- **与执行逻辑层的边界**: 控制层不实现具体业务逻辑,仅负责状态转换和任务分发;具体的任务队列实现、事件日志记录等由执行逻辑层完成


**docker-compose.yml 结构** (位于项目根目录):
```yaml
services:
  sdpj-backend:
    build:
      context: .
      dockerfile: docker/Dockerfile.backend
    volumes:
      - ./data:/app/data
      - ~/.sdpj/secret.key:/root/.sdpj/secret.key:ro  # 挂载密钥文件(只读)
    environment:
      # SDPJ_SECRET_KEY 可选: 未设置时从 ~/.sdpj/secret.key 读取
      - SDPJ_ADAPTERS_DIR=/app/data/uploads/adapters
    ports:
      - "8000:8000"

  sdpj-frontend:
    build:
      context: .
      dockerfile: docker/Dockerfile.frontend
    ports:
      - "3000:80"
    depends_on:
      - sdpj-backend
```

**Docker 部署密钥管理**:
- 容器内密钥文件路径: `/root/.sdpj/secret.key` (Linux 容器)
- 通过 volume 挂载宿主机密钥文件,避免在镜像中硬编码
- 首次部署时,宿主机需先运行一次应用生成密钥,或手动创建密钥文件

## 十、部署模式目录差异

**Monorepo 策略说明**:

本项目采用 **Monorepo (单一代码仓库)** 策略,将前端 (`ui/webui/frontend/`) 和后端 (`sdpj/`) 代码放在同一仓库中管理,原因如下:

1. **版本同步**: 前后端 API 契约变更时可在同一 commit 中同步修改,避免版本不一致
2. **构建简化**: Docker Compose 可在单一仓库中编排前后端服务,无需跨仓库依赖
3. **开发便利**: 开发者可在单一仓库中进行全栈开发,减少仓库切换成本

**构建依赖关系**:
- **CLI 模式**: 
  - 仅构建 `sdpj/` 包,`ui/webui/frontend/` 目录不参与打包 (PyInstaller 排除)
  - 不需要 Docker
  - 适用场景: 纯命令行使用,单用户本地环境
  
- **WebUI 模式**: 
  - 前端: `ui/webui/frontend/` 独立构建为静态文件 (`npm run build` → `dist/`)
  - 后端: `sdpj/` 独立构建为 Python 应用
  - 两者通过 Docker Compose 或 Nginx 反向代理集成,无代码级依赖
  - **需要 Docker**: 使用 Docker Compose 一键部署前后端服务
  - 适用场景: 需要 Web 界面的多用户场景

### 10.1 CLI 模式 (单体应用)

```
部署包结构:
sdpj-cli/
├── sdpj                            # PyInstaller 打包的可执行文件
├── data/                           # 数据目录
├── .env                            # 环境配置
└── README.txt
```

### 10.2 WebUI 模式 (前后端分离)

```
部署包结构:
sdpj-webui/
├── backend/
│   ├── sdpj/                       # Python 应用
│   ├── data/
│   └── .env
├── frontend/
│   └── dist/                       # Vite 打包的静态文件
```
