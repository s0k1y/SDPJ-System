CLI / 命令行交互模块
用户交互界面，展示操作指令与系统信息，传参指令与信息，无业务逻辑
具体职责:

# 检测交互
1. 接收用户在终端输入的检测启动指令并转交系统
   - 输入:用户在命令行键入的检测命令及其参数(检测类型(静态/动态)、用户选用的检测数据集集合或私有检测配置标识、目标被测大模型标识、动态算法所需的最大迭代次数等)
   - 输出:向用户回显已受理的检测任务组标识与各子任务标识;后续以文本流形式向用户呈现任务组的接收回执
   - 触发场景:用户通过 CLI 启动 SDPJ 静态/动态自检测算法或下发新的检测任务(对应 1.spec.md 功能 1.1 / 1.2 / 1.4)
   - 调用下层:StateScheduler 的「接收并解析检测启动指令并下发为可执行任务组」
   - 不负责的边界:不做检测算法的具体执行(由 SDPJDetector 完成);不做用户意图到任务组的解析(由 StateScheduler 完成);不做被测大模型/数据集存在性的业务校验(由 StateScheduler 经下层完成)

2. 接收用户的"选择检测数据集"指令并转交系统
   - 输入:用户在命令行键入的数据集选择指令及其参数(目标数据集标识列表或私有检测配置标识)
   - 输出:向用户回显已选数据集清单的文本视图
   - 触发场景:用户在启动检测前选择要使用的检测数据集(对应 1.spec.md 功能 1.3)
   - 调用下层:StateScheduler 的「调度用户私有检测配置的全生命周期管理」(读取已选检测配置作为本次检测的输入来源)
   - 不负责的边界:不做数据集内容的加载与解析(由 PrivateConfigManager 经 DataProcessor 完成);不做数据集元信息的真实性校验

3. 展示当前检测任务的排队与执行进度
   - 输入:用户在命令行键入的进度查询指令(可附带队列任务标识或省略以查询整体队列视图)
   - 输出:以文本/字符画形式向用户呈现单个任务的当前状态或当前队列中全部任务的状态视图
   - 触发场景:用户查看检测任务排队与进度状况(对应 1.spec.md 功能 1.4 / 2.1)
   - 调用下层:StateScheduler 的「查询检测任务执行进度」
   - 不负责的边界:不做进度数据的二次计算与汇总(由 StateScheduler 完成)

# 报告交互
4. 接收用户的"生成检测报告"指令并转交系统
   - 输入:用户在命令行键入的报告生成指令及其参数(已完成的检测任务组ID、检测类型(静态/动态))
   - 输出:向用户回显已受理的报告生成回执
   - 触发场景:用户在某检测任务组所有子任务结束后请求生成报告(对应 1.spec.md 功能 1.5)
   - 调用下层:StateScheduler 的「生成检测报告」
   - 不负责的边界:不做报告统计指标的具体计算(由 ReportManager 完成);不做合规判断(由 SDPJDetector 完成)

5. 展示检测报告内容与列表
   - 输入:用户在命令行键入的报告查看/查询指令(任务组ID 查看单份;过滤条件查询列表)
   - 输出:以文本视图(含表格、字符画图表)向用户呈现单份报告的完整内容或匹配条件的报告元信息列表
   - 触发场景:用户浏览历史检测报告或查看某份报告详情(对应 1.spec.md 功能 1.5 / 1.6)
   - 调用下层:StateScheduler 的「查看与查询检测报告」
   - 不负责的边界:不做报告内容的业务解读(由用户自行判断);不做用户对报告的访问权判定(由 DACManager 经 StateScheduler 在调用前完成)

6. 接收用户的"删除检测报告"指令并转交系统
   - 输入:用户在命令行键入的报告删除指令及其参数(任务组ID / 任务ID / 检测报告ID / 结果数据ID 任选其一,以及调用者用户ID)
   - 输出:向用户回显删除结果
   - 触发场景:用户删除某份历史报告或精修报告内容(对应 1.spec.md 功能 1.6)
   - 调用下层:StateScheduler 的「删除检测报告」
   - 不负责的边界:不做删除前置访问权判定(由 StateScheduler 在调用 DACManager 前完成)

7. 接收用户的"导出检测报告"指令并向用户落地报告文件
   - 输入:用户在命令行键入的导出指令及其参数(任务组ID、目标导出文件格式)
   - 输出:在用户终端所在文件系统的目标路径上落地报告文件,并以文本反馈导出结果
   - 触发场景:用户下载历史检测报告(对应 1.spec.md 功能 1.5 / 1.6)
   - 调用下层:StateScheduler 的「导出检测报告与可视化数据准备」
   - 不负责的边界:不做报告文件序列化的具体实现(由 ReportManager 经 DataProcessor 完成)

# 系统状态与日志交互
8. 向用户实时反馈系统状态变化与执行进度
   - 输入:由 StateScheduler 推送至本模块的状态变化与执行进度反馈
   - 输出:以终端文本形式向用户即时呈现当前系统状态与进度变化
   - 触发场景:任一指令执行过程中或检测任务推进时(对应 1.spec.md 功能 2.1 / 2.1.7.1)
   - 调用下层:StateScheduler 的「维护系统当前状态并向上层反馈进度与事件」(本模块作为反馈接收端)
   - 不负责的边界:不持有系统状态本身(由 StateScheduler 持有);不做状态机状态的解释计算

9. 展示系统日志查询结果
   - 输入:用户在命令行键入的日志查询指令及其过滤条件(日志类别、时间范围、来源模块、用户ID 等,可选)
   - 输出:以文本视图(含表格、分页文本)向用户呈现匹配条件的日志条目列表
   - 触发场景:运维或用户查看系统运行状况(对应 1.spec.md 功能 2.1.7.2)
   - 调用下层:StateScheduler 的「查询系统日志」
   - 不负责的边界:不做日志内容的脱敏(由 StateScheduler 在传入参数前完成);不做日志写入(由 StateScheduler 经 EventLogger 完成)

10. 向用户呈现系统异常提示
    - 输入:由 StateScheduler 在异常恢复后向上层反馈的错误信息
    - 输出:以终端文本(含错误等级标识)向用户呈现错误信息与恢复后的系统状态
    - 触发场景:大模型 API 调用不可恢复错误、数据库访问错误、权限校验失败、文件格式校验失败等异常事件(对应 1.spec.md 功能 2.1.7.3)
    - 调用下层:StateScheduler 的「处理业务执行过程中的异常并按状态转移规则恢复」(本模块作为异常反馈接收端)
    - 不负责的边界:不做异常的业务分类与重试决策(由 StateScheduler 协调下层完成)

# 账号交互
11. 接收用户的注册与登录指令并转交系统
    - 输入:用户在命令行键入的账号、密码(明文,因 CLI 本地运行无 C-S 加密链路)
    - 输出:以终端文本向用户呈现注册结果(附用户ID)或登录结果(附用户ID 与登录会话状态)
    - 触发场景:用户通过 CLI 发起注册或登录(对应 1.spec.md 功能 3.1.1.1)
    - 调用下层:StateScheduler 的「调度用户注册与登录」
    - 不负责的边界:不做账号格式/密码强度的校验规则(由 AccountManager 在前置校验时完成);不做密码哈希(由 UserCenter 完成)

12. 接收用户的账号管理指令并转交系统
    - 输入:用户在命令行键入的账号管理指令及其参数(密码修改/账号切换/登出/注销/查询当前用户资料/列出当前用户拥有的受控资源等操作类型与必要参数)
    - 输出:以终端文本向用户呈现对应操作的结果
    - 触发场景:用户在 CLI 中操作个人账号(对应 1.spec.md 功能 3.1.1.2)
    - 调用下层:StateScheduler 的「调度账号管理操作」
    - 不负责的边界:不持久化登录会话(登录会话由 AccountManager 在内存中维护);不做密码哈希

# 权限授权交互
13. 接收用户的"授权管理"指令并转交系统
    - 输入:用户在命令行键入的授权指令及其参数(授予读权限/移除读权限/查询资源授权清单等操作类型,以及资源ID、被授权用户ID、访问控制项ID 等参数,以及调用者用户ID)
    - 输出:以终端文本向用户呈现操作结果(新创建访问控制项ID、移除结果、授权清单等)
    - 触发场景:资源拥有者通过 CLI 管理其私有资源的访问授权(对应 1.spec.md 功能 3.1.2)
    - 调用下层:StateScheduler 的「调度自主访问控制相关操作」
    - 不负责的边界:不做访问控制规则的实现(由 DACManager 完成);不做权限的传递性校验(由 DACManager 在边界声明)

# 私有资源交互
14. 接收用户的"私有检测配置管理"指令并转交系统
    - 输入:用户在命令行键入的私有配置管理指令及其参数(创建/读取/修改/删除/列出/导入/导出等操作类型,及其所需配置内容或文件路径)
    - 输出:以终端文本向用户呈现操作结果(新建配置ID、配置内容、修改/删除结果、配置清单、导出文件路径、导入后新配置ID 等)
    - 触发场景:用户在 CLI 中操作私有检测配置(对应 1.spec.md 功能 3.1.1.2);也作为启动检测前加载用户检测配置的间接入口(对应 1.spec.md 功能 1.1 / 1.2 / 1.3)
    - 调用下层:StateScheduler 的「调度用户私有检测配置的全生命周期管理」
    - 不负责的边界:不做配置文件格式校验(由 PrivateConfigManager 经 DataProcessor 完成);不做私有配置内容的存储(由 PrivateConfigManager 经 UserCenter 完成);CLI 本地运行无 C-S 链路,不涉及配置文件的传输加解密

15. 接收用户的"私有大模型适配器与私有数据集管理"指令并转交系统
    - 输入:用户在命令行键入的适配器/数据集管理指令及其参数(上传/移除适配器、上传/移除数据集或样本等操作类型,及其所需文件路径、目标大模型标识、数据集元信息、样本条目清单、所属用户ID 等)
    - 输出:以终端文本向用户呈现操作结果(新登记大模型标识、新数据集ID、各样本ID、移除结果等)
    - 触发场景:用户在 CLI 中管理私有大模型适配器或私有数据集(对应 1.spec.md 功能 3.1.1.2;数据集还可作为 1.spec.md 功能 1.3 的输入来源)
    - 调用下层:StateScheduler 的「调度用户私有大模型适配器与私有数据集的上传与移除」
    - 不负责的边界:不做适配器入库注册的具体实现(由 PrivateConfigManager 经 LLMRegistry 完成);不做数据集/样本的具体落盘(由 PrivateConfigManager 经 DataProcessor 完成)

# 终端会话与帮助
16. 提供命令提示与帮助信息
    - 输入:用户在命令行键入的帮助指令(全局帮助或子命令帮助)
    - 输出:以终端文本向用户呈现可用命令清单、参数说明与使用示例
    - 触发场景:用户在 CLI 中查阅可用命令与用法
    - 调用下层:无(本职责仅为静态用法说明的呈现,不触发系统业务调度)
    - 不负责的边界:不做指令的实际执行(由其余职责按指令类型分别转交)

17. 维持本地终端用户会话
    - 输入:本模块本地维持的"当前已登录用户"上下文(由职责11登录后产生)
    - 输出:在后续指令转交 StateScheduler 时附带"当前调用者用户ID"以驱动归属判定;用户登出/账号切换/注销后清空本地上下文
    - 触发场景:用户登录后在同一 CLI 会话中连续下发指令期间(对应 1.spec.md 功能 2.1.7.1 / 3.1.1.1 / 3.1.1.2)
    - 调用下层:无(本职责仅维持本模块本地的会话上下文,不直接触发下层业务调度;真正的会话生命周期由 AccountManager 在内存中维护)
    - 不负责的边界:不做登录会话的持久化(由 AccountManager 在内存中维护);不做权限判定(由 DACManager 经 StateScheduler 完成)

# 接口契约
18. 本模块不向系统其他模块对外暴露接口;仅作为用户交互入口,通过 StateSchedulerInterface 与系统交互(符合 4.模型依赖关系图.puml 中 CLI → StateScheduler 边)

不需要的:[做任何业务逻辑(检测算法/报告统计/合规判断/权限规则等均不属于 UI 层),做密码哈希,做用户对资源的访问权判定,做检测样本/PoC 的语义校验,做日志/报告内容的脱敏,持久化登录会话到外部存储(由 AccountManager 在内存中维护),做 C-S 加解密(CLI 本地运行无网络通信,无加解密需求),越层调用控制层以下任何模块(仅可调用 StateScheduler 的对外能力)]
依赖模块:StateScheduler,调用接口:StateSchedulerInterface
应实现接口:无
被依赖模块:无
技术细节:(暂定)
