SecureCommManager / 安全通信管理模块
具体职责:

# 敏感数据加密
1. 对账号密码进行加密
   - 输入:明文账号密码
   - 输出:密文
   - 触发场景:C-S 模式下账号密码在通信链路上传输前的加密处理(对应 1.spec.md 功能 2.1.8.2),涉及用户注册/登录/修改密码/账号切换等环节(对应 1.spec.md 功能 3.1.1.1 / 3.1.1.2)
   - 调用下层:无
   - 不负责的边界:不做账号密码的合法性校验(由 AccountManager 在业务层完成);加解密所需密钥在本模块内部维护,不对外暴露

2. 对用户私有检测配置文件进行加密
   - 输入:明文形式的私有检测配置文件内容
   - 输出:密文
   - 触发场景:C-S 模式下私有检测配置文件在通信链路上传输前的加密处理(对应 1.spec.md 功能 2.1.8.2),涉及私有检测配置的上传/下载/导入/导出等环节(对应 1.spec.md 功能 3.1.1.2)
   - 调用下层:无
   - 不负责的边界:不做文件格式校验与业务语义审查(由 PrivateConfigManager 在业务层协调完成);不做文件内容的落盘

# 敏感数据解密
3. 对账号密码进行解密
   - 输入:密文形式的账号密码
   - 输出:明文
   - 触发场景:C-S 模式下接收到密文账号密码后的解密处理(对应 1.spec.md 功能 2.1.8.2)
   - 调用下层:无
   - 不负责的边界:不做解密后明文的下一步业务处理(由 AccountManager 等调用方完成);不做密码哈希(由 UserCenter 借 UtilsLib 完成)

4. 对用户私有检测配置文件进行解密
   - 输入:密文形式的私有检测配置文件内容
   - 输出:明文
   - 触发场景:C-S 模式下接收到密文私有检测配置文件后的解密处理(对应 1.spec.md 功能 2.1.8.2)
   - 调用下层:无
   - 不负责的边界:不做解密后文件内容的下一步业务处理(由 PrivateConfigManager 等调用方完成)

# 接口契约
5. 通过 SecureCommManagerInterface 对外暴露上述能力,被 StateScheduler 调用(符合 4.模型依赖关系图.puml 中 StateScheduler → SecureCommManager 边)

不需要的:[建立/维护 C-S 底层通信链路(由 UI 层与 StateScheduler 间的网络通道承载),对账号密码与私有检测配置文件之外的对象做加解密,向外部模块分发或暴露密钥,对解密后的明文做业务语义处理,记录加解密操作日志(由 EventLogger 完成)]

依赖模块:无 调用接口:无
应实现接口:SecureCommManagerInterface
被依赖模块:StateScheduler
技术细节:(暂定)
本模块自包含,不调用本系统任何其他模块;仅可使用 Python 标准库与第三方库(具体库待 5.项目技术选型 阶段确定)。下层持久化 / 加密 / 队列状态管理等能力由模块内部实现。
本模块通过与StateScheduler进行信息流和控制流的交换来使得本模块能够正常运行,而不依赖任何下层设施。
