DACManager / 自主访问控制管理模块
具体职责:

# 访问权授予
1. 授予其他用户对私有资源的读权限
   - 输入:资源ID、被授权用户ID、调用者用户ID
   - 输出:新创建的访问控制项ID
   - 触发场景:资源拥有者(私有检测配置文件/私有数据集/私有大模型适配器等的拥有者)将读权限授予其他用户(对应 1.spec.md 功能 3.1.2)
   - 调用下层:UserCenter 的「按 ID 查询资源」确认资源存在并取得拥有者用户ID,在确认调用者即为资源拥有者后,调用 UserCenter 的「授予他人访问权」
   - 错误情形:调用者非资源拥有者时拒绝授予;资源ID/被授权用户ID 不存在由下层透传错误;同 (资源ID, 被授权用户ID) 组合已存在时下层幂等返回
   - 不负责的边界:不维护受控资源的具体内容(由 PrivateConfigManager 等业务模块完成);被授予使用权限的用户不具备任何权限再授予能力(由本模块在调用者校验时强制)

# 访问权移除
2. 移除已授予的读权限
   - 输入:访问控制项ID、调用者用户ID
   - 输出:移除结果
   - 触发场景:资源拥有者收回先前对某用户授予的读权限(对应 1.spec.md 功能 3.1.2)
   - 调用下层:UserCenter 的「移除已授予的访问权」(在校验调用者为对应资源拥有者后执行)
   - 错误情形:调用者非对应资源拥有者时拒绝移除

# 访问权判定
3. 判定用户对资源是否具备访问权
   - 输入:资源ID、用户ID
   - 输出:是否具备访问权的布尔判定
   - 判定规则:资源拥有者默认具备读权限与移除权限;在 ACL 中被登记为被授权用户的具备读权限;其他用户不具备访问权
   - 触发场景:用户对私有文件发起使用请求前的前置访问权校验(对应 1.spec.md 功能 3.1.2);PrivateConfigManager / SDPJDetector 在加载私有资源前的前置校验
   - 调用下层:UserCenter 的「按 ID 查询资源」获得拥有者;UserCenter 的「判定用户对资源是否具备访问权」校验 ACL

# 授权状况查询
4. 查询某资源的授权清单
   - 输入:资源ID、调用者用户ID
   - 输出:该资源下全部访问控制项(每条含访问控制项ID、被授权用户ID)
   - 触发场景:资源拥有者查看其资源当前的授权状况(对应 1.spec.md 功能 3.1.2)
   - 调用下层:UserCenter 的「查询某资源的授权清单」(在校验调用者为资源拥有者后执行)
   - 错误情形:调用者非资源拥有者时拒绝查询

# 接口契约
5. 通过 DACManagerInterface 对外暴露上述能力,被 StateScheduler 调用(符合 4.模型依赖关系图.puml 中 StateScheduler → DACManager 边)

不需要的:[实现权限委托机制,记录权限变更日志(由 EventLogger 完成),支持权限继承,允许被授权用户再授权,维护受控资源的具体内容,做账号与凭据管理(由 AccountManager 与 UserCenter 完成),为私有资源之外的对象做权限管理]

依赖模块:UserCenter,调用接口:UserCenterInterface
应实现接口:DACManagerInterface
被依赖模块:StateScheduler
技术细节:(暂定)通过自主访问控制列表（ACL）来管理用户对资源的访问权限
