AccountManager / 用户账号管理模块
具体职责:

# 账号注册与注销
1. 用户注册
   - 输入:账号、明文密码
   - 输出:新注册用户的用户ID
   - 触发场景:用户通过 UI 层发起注册请求(对应 1.spec.md 功能 3.1.1.1)
   - 调用下层:UserCenter 的「注册新用户」
   - 错误情形:账号格式不合法/密码强度不足在本模块做前置校验后拒绝;账号已存在由下层透传错误
   - 不负责的边界:不做密码哈希(由 UserCenter 借 UtilsLib 完成)

2. 用户注销
   - 输入:用户ID
   - 输出:注销结果
   - 后置条件:本模块维护的对应登录会话被清除;下层级联清理该用户拥有的资源与相关 ACL 条目
   - 触发场景:用户在账号管理中选择注销账号(对应 1.spec.md 功能 3.1.1.2)
   - 调用下层:UserCenter 的「注销用户」

# 登录与会话维护
3. 用户登录
   - 输入:账号、明文密码
   - 输出:登录结果(通过时附带用户ID)
   - 后置条件:通过时建立本模块维护的登录会话,记录当前登录态与用户ID
   - 触发场景:用户通过 UI 层发起登录(对应 1.spec.md 功能 3.1.1.1)
   - 调用下层:UserCenter 的「登录凭据校验」
   - 不负责的边界:不做密码哈希(由 UserCenter 完成)

4. 维护登录会话与在线态
   - 输入:无(本模块在登录成功时主动建立、登出/账号切换/注销时清理)
   - 输出:当前登录用户ID或未登录态
   - 触发场景:任一需要识别"当前登录用户"的业务在调用前查询当前会话(支撑 1.spec.md 功能 3.1.1.2 / 1.5 / 1.6 / 3.1.2 等)
   - 不负责的边界:登录会话维护是 AccountManager 的本模块职责,UserCenter 不持有会话态(参见 UserCenter「不需要的」边界声明)

5. 用户登出
   - 输入:无
   - 输出:登出结果
   - 后置条件:当前登录会话被清除
   - 触发场景:用户主动登出或账号切换的前置步骤

# 账号管理
6. 修改指定用户密码
   - 输入:用户ID、原明文密码、新明文密码
   - 输出:修改结果
   - 触发场景:用户在账号管理中修改密码(对应 1.spec.md 功能 3.1.1.2)
   - 调用下层:UserCenter 的「登录凭据校验」核对原密码;再调用 UserCenter 的「修改用户密码」写入新密码
   - 错误情形:原密码不匹配时拒绝更新;新密码不满足强度要求在本模块前置校验后拒绝
   - 安全说明:user_id 由调用方（StateScheduler）从请求上下文注入，不依赖本模块内部单例状态，避免多用户并发时身份混淆

7. 账号切换
   - 输入:目标账号、明文密码
   - 输出:切换结果(通过时附带新登录用户ID)
   - 后置条件:清除旧登录会话,建立新登录会话
   - 触发场景:用户在账号管理中切换登录账号(对应 1.spec.md 功能 3.1.1.2)
   - 实现方式:仅调用 login() 完成凭据校验与会话建立，不单独查询用户信息

8. 查询指定用户资料
   - 输入:用户ID
   - 输出:指定用户的资料(用户ID、账号)
   - 触发场景:用户查看个人信息;上层模块需识别当前用户身份(对应 1.spec.md 功能 3.1.1.2)
   - 调用下层:UserCenter 的「按用户 ID 查询用户信息」
   - 安全说明:user_id 由调用方从请求上下文注入

9. 修改指定用户用户名
   - 输入:用户ID、新用户名
   - 输出:修改结果
   - 触发场景:用户在账号管理中修改用户名(对应 1.spec.md 功能 3.1.1.2)
   - 调用下层:UserCenter 的「修改用户名」
   - 安全说明:user_id 由调用方从请求上下文注入

10. 列出指定用户拥有的受控资源
   - 输入:用户ID
   - 输出:指定用户拥有的全部受控资源列表(每条含资源ID、资源类型)
   - 触发场景:用户在账号管理中查看其私有检测配置文件、私有数据集、私有大模型适配器等受控资产(对应 1.spec.md 功能 3.1.1.2)
   - 调用下层:UserCenter 的「按拥有者查询资源清单」
   - 安全说明:user_id 由调用方从请求上下文注入
   - 不负责的边界:不做受控资源内容管理(由 PrivateConfigManager 完成);不做授权判定(由 DACManager 完成)

11. 列出分享给指定用户的资源
   - 输入:用户ID
   - 输出:通过 DAC 授权给该用户的全部受控资源列表(每条含资源ID、资源类型、资源拥有者信息)
   - 触发场景:用户查看他人分享给自己的资源列表(对应 1.spec.md 功能 3.1.2)
   - 调用下层:UserCenter 的「按被授权用户查询资源清单」
   - 安全说明:user_id 由调用方从请求上下文注入

12. 列出系统中全部用户
   - 输入:无(或管理员凭证)
   - 输出:系统中全部注册用户的列表(每条含用户ID、账号)
   - 触发场景:管理员查看系统用户列表；DAC 授权时查询可授权的目标用户
   - 调用下层:UserCenter 的「列出全部用户」

# 接口契约
13. 通过 AccountManagerInterface 对外暴露上述能力,被 StateScheduler 调用(符合 4.模型依赖关系图.puml 中 StateScheduler → AccountManager 边)

不需要的:[做密码哈希(由 UserCenter 完成),做账号格式与密码强度的具体规则规范(可在本模块做前置校验,但规则的工程化沉淀属于配置层),做受控资源内容管理(由 PrivateConfigManager 完成),做权限授予与判定(由 DACManager 完成),持久化登录会话到外部存储(仅本模块内存态)]

依赖模块:UserCenter,调用接口:UserCenterInterface
应实现接口:AccountManagerInterface
被依赖模块:StateScheduler
技术细节:(暂定)
