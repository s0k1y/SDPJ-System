具体职责:

# 适配器制品管理
> **存储说明**：适配器制品使用进程内存字典（`_adapters` / `_instances` / `_metadata`）存储，不做持久化。进程重启后所有已注册的适配器将丢失。

1. 注册私有大模型 API 适配器（存储于进程内存中，进程重启后丢失）
   - 输入:适配器配置文件（JSON，包含 api_key、base_url、model、request_format、timeout 等参数）、目标大模型标识
   - 输出:注册结果及其对应创建好的 API 调用服务实例句柄
   - 错误情形:适配器配置不符合大模型服务接口规范时拒绝注册;同标识适配器已存在时拒绝写入
   - 触发场景:用户上传私有大模型适配器配置（对应 1.spec.md 功能 3.1.1.2）
   - 不负责的边界:不维护适配器与用户/私有配置的绑定关系(由 LLMRegistry 维护);不做用户权限判定
   - **向上抽象职责**:将不同规格的 JSON 配置（OpenAI 格式、Anthropic 格式等）转换为系统内部统一的抽象接口参数，屏蔽底层 API 差异，向上层提供统一调用接口

2. 移除已注册的私有大模型 API 适配器
   - 输入:目标大模型标识
   - 输出:移除结果
   - 后置条件:对应适配器服务实例被销毁;适配器制品从内存中清除
   - 触发场景:用户移除私有大模型适配器(对应 1.spec.md 功能 3.1.1.2)

3. 列出已注册的适配器
   - 输入:无
   - 输出:已注册适配器清单(每条含大模型标识、适配器元信息)
   - 触发场景:LLMRegistry 同步注册中心视图

4. 按大模型标识查询适配器
   - 输入:大模型标识
   - 输出:对应适配器的元信息;不存在时通过明确错误反馈
   - 触发场景:调用方在创建服务实例前判断适配器是否可用

# 服务实例管理
5. 获取大模型 API 调用服务实例
   - 输入:大模型标识
   - 输出:对应适配器创建好的 API 调用服务实例
   - 错误情形:对应适配器尚未注册时无法创建实例,返回明确错误
   - 触发场景:LLMService 在检测执行中对目标大模型发起调用(对应 1.spec.md 功能 1.1 / 1.2)
   - 不负责的边界:不做服务实例调用过程中的业务编排(由 LLMService 处理)

6. 销毁大模型 API 调用服务实例（同步方法，仅做内存清理，无 I/O）
   - 输入:服务实例句柄
   - 输出:销毁结果
   - 触发场景:对应适配器被移除时;调用方明确释放资源时

# 错误语义标准化
7. 统一大模型 API 调用错误
   - 输入:底层适配器或第三方 SDK 抛出的原生异常
   - 输出:语义标准化后的错误结果(分类：网络错误、鉴权错误、参数错误、限流错误、模型内部错误、超时错误、未知错误)
   - 触发场景:任一适配器在实际发起大模型 API 调用过程中发生异常,需向调用方反馈可识别的错误语义(支撑 1.spec.md 功能 1.1 / 1.2 的异常处理路径)
   - 不负责的边界:不做重试/退避决策(由 LLMService 处理);不做鉴权凭据管理

# 接口契约
8. 通过 LLMAdapterInterface 对外暴露上述能力,LLMService 与 LLMRegistry 为调用方(符合 4.模型依赖关系图.puml 中 LLMService → LLMAdapterLib 与 LLMRegistry → LLMAdapterLib 两条边)

依赖模块:无,调用接口:无

技术细节:(暂定)
大模型注册中心与大模型API适配库采用了面向对象编程的思想进行设计。
大模型API适配库模块由多个不同大模型的API适配器组成，适配器将不同大模型API调用中的具体调用方式适配于大模型服务接口的抽象方法，在API适配器本质上是在大模型API调用文件的进一步封装其内部调用的技术细节。

用户可以通过三种方式接入大模型：
1. **使用内置规范**：提供包含 request_format（openai/anthropic）、api_key、base_url、model、timeout 等参数的 JSON 配置文件，系统根据 request_format 字段选择对应的内置适配器类（OpenAIAdapter、AnthropicAdapter）并创建实例
2. **使用系统抽象规范**：用户编写自定义适配器类，继承 LLMAdapter 基类并实现统一的 call() 方法（接收 prompt、system_prompt、max_tokens、temperature、timeout 参数，返回统一格式的响应），然后提供包含 request_format="custom"、adapter_class（自定义适配器类的完全限定名）及其他配置参数的 JSON 配置文件
3. **扩展内置规范**：为新的 API 格式（如 Google、Cohere）添加新的内置适配器类

适配器的核心职责是将不同厂商的 API 格式（请求参数、响应结构）转换为系统内部统一的抽象接口，实现"向上抽象"，使上层调用方无需关心底层 API 差异。配置参数（api_key、base_url、model）在适配器构造时确定，调用时只传递业务参数（prompt、system_prompt 等），不允许运行时覆盖配置。

timeout 参数说明：HTTP 请求超时时间，单位秒，默认 60 秒，用于防止网络请求长时间挂起。当大模型服务响应慢或网络不稳定时，超过 timeout 时间后会主动断开连接并抛出超时错误。

对大模型API适配器进行调用将返回一个API调用服务实例。
大模型API适配库在收到大模型注册中心上传指令后，将私有大模型API适配器配置注册到内存后通知大模型注册中心并返回一个对应适配器创建好的服务实例；在收到大模型注册中心移除指令后，将对应适配器服务实例销毁，从内存中删除对应适配器配置并通知大模型注册中心。
连接复用：每个适配器实例将 aiohttp.ClientSession 作为实例属性维护（而非每次调用新建），并提供 close() 方法供生命周期管理，实现 HTTP 连接池复用，避免频繁建立/销毁连接的开销。
