LLMAdapterLib / 大模型API适配库模块
具体职责:

# 适配器制品管理
1. 入库私有大模型 API 适配器
   - 输入:适配器代码/文件、目标大模型标识
   - 输出:入库结果及其对应创建好的 API 调用服务实例句柄
   - 错误情形:适配器不符合大模型服务接口规范时拒绝入库;同标识适配器已存在时拒绝写入
   - 触发场景:用户上传私有大模型适配器(对应 1.spec.md 功能 3.1.1.2)
   - 不负责的边界:不维护适配器与用户/私有配置的绑定关系(由 LLMRegistry 维护);不做用户权限判定

2. 移除已入库的私有大模型 API 适配器
   - 输入:目标大模型标识
   - 输出:移除结果
   - 后置条件:对应适配器服务实例被销毁;适配器制品从库中清除
   - 触发场景:用户移除私有大模型适配器(对应 1.spec.md 功能 3.1.1.2)

3. 列出已入库的适配器
   - 输入:无
   - 输出:已入库适配器清单(每条含大模型标识、适配器元信息)
   - 触发场景:LLMRegistry 同步注册中心视图

4. 按大模型标识查询适配器
   - 输入:大模型标识
   - 输出:对应适配器的元信息;不存在时通过明确错误反馈
   - 触发场景:调用方在创建服务实例前判断适配器是否可用

# 服务实例管理
5. 获取大模型 API 调用服务实例
   - 输入:大模型标识
   - 输出:对应适配器创建好的 API 调用服务实例
   - 错误情形:对应适配器尚未入库时无法创建实例,返回明确错误
   - 触发场景:LLMService 在检测执行中对目标大模型发起调用(对应 1.spec.md 功能 1.1 / 1.2)
   - 不负责的边界:不做服务实例调用过程中的业务编排(由 LLMService 处理)

6. 销毁大模型 API 调用服务实例（同步方法，仅做内存清理，无 I/O）
   - 输入:服务实例句柄
   - 输出:销毁结果
   - 触发场景:对应适配器被移除时;调用方明确释放资源时

# 错误语义标准化
7. 统一大模型 API 调用错误
   - 输入:底层适配器或第三方 SDK 抛出的原生异常
   - 输出:语义标准化后的错误结果(分类：网络错误、鉴权错误、参数错误、限流错误、模型内部错误、超时错误、未知错误)
   - 触发场景:任一适配器在实际发起大模型 API 调用过程中发生异常,需向调用方反馈可识别的错误语义(支撑 1.spec.md 功能 1.1 / 1.2 的异常处理路径)
   - 不负责的边界:不做重试/退避决策(由 LLMService 处理);不做鉴权凭据管理

# 接口契约
8. 通过 LLMAdapterInterface 对外暴露上述能力,LLMService 与 LLMRegistry 为调用方(符合 4.模型依赖关系图.puml 中 LLMService → LLMAdapterLib 与 LLMRegistry → LLMAdapterLib 两条边)

不需要的:[管理 API 密钥（密钥由调用方注入，适配器仅透传使用，不做存储/轮换/分发）,处理 API 版本兼容,做调用方的重试/退避决策,维护适配器与用户/私有配置的绑定关系,做用户权限判定,做检测业务编排]
依赖模块:无,调用接口:无
应实现接口:LLMAdapterInterface
被依赖模块:LLMService,LLMRegistry
技术细节:(暂定)
大模型注册中心与大模型API适配库采用了面向对象编程的思想进行设计。
大模型API适配库模块由多个不同大模型的API适配器组成，适配器将不同大模型API调用中的具体调用方式适配于大模型服务接口的抽象方法，在API适配器本质上是在大模型API调用文件的进一步封装其内部调用的技术细节。用户应在其大模型API调用文件的基础上遵循本系统大模型服务接口规范写出对应适配器并上传。对大模型API适配器进行调用将返回一个API调用服务实例。
大模型API适配库在收到大模型注册中心上传指令后，将私有大模型API适配器入库后通知大模型注册中心并返回一个对应适配器创建好的服务实例；在收到大模型注册中心移除指令后，将对应适配器服务实例销毁，删除对应适配器并通知大模型注册中心。
