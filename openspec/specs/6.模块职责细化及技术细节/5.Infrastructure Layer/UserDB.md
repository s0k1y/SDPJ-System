UserDB / 用户信息数据库模块
具体职责:

# 用户账号级能力
1. 创建用户
   - 输入:账号、密码(已由调用方完成加密/哈希处理)
   - 输出:新创建用户的用户 ID
   - 错误情形:账号在系统内重复时拒绝写入
   - 触发场景:用户注册(对应 1.spec.md 功能 3.1.1.1)
   - 不负责的边界:不做密码加密/哈希(由 UserCenter 借助 UtilsLib 完成);不做账号格式与密码强度校验(由调用方完成)

2. 删除用户
   - 输入:用户ID
   - 输出:删除结果
   - 后置条件:级联删除该用户拥有的资源记录及其相关 ACL 项(作为拥有者);并清理该用户作为被授权者出现在他人资源 ACL 中的所有条目
   - 触发场景:用户注销账号(对应 1.spec.md 功能 3.1.1.2)

3. 更新用户密码
   - 输入:用户ID、新密码(已由调用方完成加密/哈希处理)
   - 输出:更新结果
   - 触发场景:用户修改密码(对应 1.spec.md 功能 3.1.1.2)

4. 按账号查询用户（get_user_by_username）
   - 输入:账号
   - 输出:用户信息（用户ID、账号、已存储密码、created_at）
   - 触发场景:用户登录时根据账号读取凭据用于校验(对应 1.spec.md 功能 3.1.1.1);账号切换(对应 1.spec.md 功能 3.1.1.2)

5. 按 ID 查询用户（get_user_by_id）
   - 输入:用户ID
   - 输出:用户信息（用户ID、账号、created_at），不包含 password 字段
   - 触发场景:AccountManager 通过 UserCenter 读取当前用户信息(对应 1.spec.md 功能 3.1.1.2)

19. 获取所有用户（get_all_users）
    - 输入:无
    - 输出:所有用户列表（每条含 user_id、username、created_at），不包含 password 字段
    - 触发场景:管理功能列举全部注册用户

# 资源级能力
6. 登记资源
   - 输入:资源类型、资源拥有者用户ID
   - 输出:新创建资源的资源 ID
   - 错误情形:资源拥有者用户ID 不存在时拒绝写入
   - 触发场景:用户创建私有检测配置文件等受控资源时为其建立 ACL 承载点(对应 1.spec.md 功能 3.1.1.2)
   - 不负责的边界:除「私有检测配置」类型资源的内容(由本模块专属的配置内容表承载)外,不维护其他资源实体的具体内容(仅维护资源元信息与访问控制)

7. 删除资源
   - 输入:资源ID
   - 输出:删除结果
   - 后置条件:级联删除该资源相关的全部访问控制项;若该资源在私有检测配置内容表中有对应条目,一并级联清理
   - 触发场景:用户删除私有检测配置文件等受控资源(对应 1.spec.md 功能 3.1.1.2)

8. 按拥有者查询资源列表
   - 输入:用户ID
   - 输出:该用户拥有的全部资源(每条含资源ID、资源类型)
   - 触发场景:AccountManager 列出当前用户的受控资产

9. 按 ID 查询资源
   - 输入:资源ID
   - 输出:资源信息(资源类型、资源拥有者用户ID);不存在时通过明确错误反馈
   - 触发场景:权限判定前定位资源的归属(对应 1.spec.md 功能 3.1.2)

# 访问控制列表级能力
10. 添加访问控制项
    - 输入:资源ID、被授权用户ID
    - 输出:新创建的访问控制项 ID
    - 错误情形:资源ID 或 被授权用户ID 不存在时拒绝写入;同 (资源ID, 被授权用户ID) 组合已存在时幂等返回
    - 触发场景:资源拥有者对其他用户授予访问权限(对应 1.spec.md 功能 3.1.2)
    - 不负责的边界:不判定调用者是否具备"授权他人"的资格(由 DACManager 在调用前完成)

11. 删除访问控制项
    - 输入:访问控制项ID
    - 输出:删除结果
    - 触发场景:资源拥有者移除对其他用户的授权(对应 1.spec.md 功能 3.1.2)

12. 按资源ID查询访问控制项列表
    - 输入:资源ID
    - 输出:该资源下全部访问控制项(每条含访问控制项ID、被授权用户ID)
    - 触发场景:DACManager 展示某资源的授权状况(对应 1.spec.md 功能 3.1.2)

13. 判定访问控制项是否存在
    - 输入:资源ID、被授权用户ID
    - 输出:对应授权项是否存在的布尔判定
    - 触发场景:DACManager 校验某用户对某资源是否具备访问权(对应 1.spec.md 功能 3.1.2)

# 私有检测配置内容级能力
14. 写入私有检测配置内容
    - 输入:配置ID(等同于其对应资源的资源ID)、配置内容JSON
    - 输出:写入结果
    - 错误情形:对应资源ID 不存在时拒绝写入;同 配置ID 已有内容时拒绝(覆盖请用更新)
    - 触发场景:用户创建一份私有检测配置文件后持久化其配置内容(对应 1.spec.md 功能 3.1.1.2)
    - 不负责的边界:不解析配置内容的业务语义;不做调用方权限判定(由 DACManager 在调用前完成)

15. 按 ID 读取私有检测配置内容
    - 输入:配置ID
    - 输出:对应配置内容JSON;不存在时通过明确错误反馈
    - 触发场景:用户查看私有检测配置;系统在启动检测前加载用户选择的检测配置(对应 1.spec.md 功能 1.1 / 1.2 / 1.3 / 3.1.1.2)

16. 更新私有检测配置内容
    - 输入:配置ID、新配置内容JSON
    - 输出:更新结果
    - 错误情形:配置ID 不存在时拒绝更新
    - 触发场景:用户编辑已有的私有检测配置(对应 1.spec.md 功能 3.1.1.2)

17. 删除私有检测配置内容
    - 输入:配置ID
    - 输出:删除结果
    - 触发场景:显式清除某条配置内容(通常由 #7 删除资源级联清理覆盖,显式删除作为可选路径)
    - 不负责的边界:级联触发由资源外键约束保证,本职责仅做单条配置内容的显式清除

# 接口契约
20. 通过 UserDBInterface 对外暴露上述能力,UserCenter 是唯一调用方(符合 4.模型依赖关系图.puml 中 UserCenter → UserDB 边)

不需要的:[对密码做加密/哈希处理,维护用户登录会话与在线态,做密码强度/账号格式校验,做权限授予资格的业务判定,做调用方业务逻辑(分页/排序/筛选),对「私有检测配置」类型之外的受控资源维护其具体内容(该类型由本模块专属的配置内容表承载,其他类型仅维护元信息与 ACL)]

依赖模块:无
应实现接口:UserDBInterface
被依赖模块:UserCenter

# 物理结构合并

UserDB 的数据表（`User`、`Resource`、`AccessControl`、`PrivateConfig`）与 SampleDB、ResultDB 的数据表合并存储于同一 SQLite 文件 `sdpj/infrastructure/database/sdpj.db`。

原因：ResultDB 的 `TaskGroup.user_id` 引用本模块的 `User.user_id`。若两模块使用独立文件，该外键无法被 SQLite 引擎强制执行（跨文件外键约束不可用），数据完整性将崩溃。合并后由数据库引擎通过 `PRAGMA foreign_keys=ON` 统一执行所有约束。

模块间逻辑隔离通过 `UserDBInterface` 接口层维持，物理合并不影响模块边界。

用户信息数据库模块:存储用户账号信息，并进一步实现自主访问控制列表。
在自主访问控制列表的基础逻辑上，设计符合数据库规范性3NF标准的关系模型，得：
用户(用户ID[主键], 账号, 密码, created_at)
资源(资源ID[主键], 资源类型, 资源拥有者用户ID[外键_用户(用户ID)], created_at)
访问控制列表(访问控制项ID[主键], 资源ID[外键_资源(资源ID)], 被授权用户ID[外键_用户(用户ID)], created_at)
私有检测配置内容(配置ID[主键/外键_资源(资源ID)], 配置内容JSON, created_at, updated_at)
