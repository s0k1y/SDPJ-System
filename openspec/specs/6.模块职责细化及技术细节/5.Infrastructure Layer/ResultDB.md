ResultDB / 检测结果数据库模块
具体职责:

# 检测任务组级能力
1. 创建检测任务组
   - 输入:用户ID、模型ID
   - 输出:新创建任务组的任务组 ID
   - 触发场景:用户启动检测执行时为本次检测开设任务组(对应 1.spec.md 功能 1.1 / 1.2)
   - 错误情形:模型ID 在被测大模型表中不存在时拒绝写入
   - 不负责的边界:不校验用户ID合法性(用户ID 是 UserDB.用户 的跨库外键,由调用方在调用前保证)

2. 删除检测任务组
   - 输入:任务组ID
   - 输出:删除结果
   - 后置条件:级联删除该任务组下的所有检测任务、检测报告与检测结果数据(满足外键完整性)
   - 触发场景:用户删除某次检测产出的报告(对应 1.spec.md 功能 1.6)

3. 查询检测任务组列表
   - 输入:过滤条件(用户ID / 模型ID,可选)
   - 输出:任务组元信息列表(任务组ID、用户ID、模型ID)
   - 触发场景:用户浏览历史检测(对应 1.spec.md 功能 1.6)
   - 不负责的边界:不做分页/排序业务逻辑(由调用方按需处理)

4. 按 ID 查询单个检测任务组
   - 输入:任务组ID
   - 输出:任务组元信息;不存在时通过明确错误反馈
   - 触发场景:ReportManager 通过 DataProcessor 定位特定任务组的检测产出(对应 1.spec.md 功能 1.5)

# 检测任务级能力
5. 创建检测任务
   - 输入:所属任务组ID、数据集ID、初始任务状态、开始时间
   - 输出:新创建任务的任务 ID
   - 错误情形:任务组ID 不存在时拒绝写入
   - 触发场景:SDPJ 静态/动态算法在每个待测数据集 D_i 上各启动一次检测(对应 1.spec.md 功能 1.1 / 1.2)
   - 不负责的边界:不校验数据集ID 是否在 SampleDB 中存在(跨库外键,由调用方在调用前保证)

6. 更新检测任务状态
   - 输入:任务ID、任务状态、结束时间(可选)
   - 输出:更新结果
   - 触发场景:任务由"进行中"转入"完成"或"异常中断"时

7. 删除检测任务
   - 输入:任务ID
   - 输出:删除结果
   - 后置条件:级联删除该任务对应的检测报告与检测结果数据
   - 触发场景:用户删除单条检测任务(对应 1.spec.md 功能 1.6)

8. 按任务组ID查询任务列表
   - 输入:任务组ID
   - 输出:该任务组下全部检测任务(每条含任务ID、数据集ID、任务状态、开始时间、结束时间)
   - 触发场景:ReportManager 汇总任务组下所有子任务以生成可视化报告(对应 1.spec.md 功能 1.5)

9. 按 ID 查询单个检测任务
   - 输入:任务ID
   - 输出:任务详情

# 被测大模型级能力
10. 登记被测大模型
    - 输入:模型ID
    - 输出:登记结果
    - 触发场景:任务组创建前,若模型尚未在被测大模型表中则先行入表
    - 错误情形:模型ID 已存在时幂等返回,不视为错误
    - 不负责的边界:不维护大模型的业务元数据(能力信息由 LLMRegistry 维护)

11. 按 ID 查询被测大模型
    - 输入:模型ID
    - 输出:被测大模型条目是否存在
    - 触发场景:创建任务组前校验模型ID合法性

# 检测报告级能力
12. 创建检测报告
    - 输入:所属任务ID
    - 输出:新创建报告的检测报告 ID
    - 错误情形:任务ID 不存在时拒绝写入
    - 触发场景:一条检测任务完成后为其生成对应报告条目(对应 1.spec.md 功能 1.1 / 1.2 的收尾步骤)

13. 删除检测报告
    - 输入:检测报告ID
    - 输出:删除结果
    - 后置条件:级联删除该报告下的全部检测结果数据
    - 触发场景:用户删除单份检测报告(对应 1.spec.md 功能 1.6)

14. 按任务ID查询检测报告
    - 输入:任务ID
    - 输出:该任务对应的报告(任务与报告为 1-1 关系)
    - 触发场景:ReportManager 定位特定任务的报告(对应 1.spec.md 功能 1.5)

15. 查询检测报告列表
    - 输入:过滤条件(任务组ID 等,可选)
    - 输出:报告元信息列表
    - 触发场景:用户浏览历史报告(对应 1.spec.md 功能 1.6)

16. 按 ID 查询单份检测报告
    - 输入:检测报告ID
    - 输出:报告元信息

# 检测结果数据级能力
17. 追加检测结果数据条目
    - 输入:检测报告ID、风险具体子类、被测大模型输出内容、合规判断结果
    - 输出:新创建条目的结果数据ID
    - 错误情形:检测报告ID 不存在时拒绝写入
    - 触发场景:检测过程中每完成对一条样本的判断后,将模型响应与合规结论写入(对应 1.spec.md 功能 1.1 / 1.2)
    - 不负责的边界:不做合规判断逻辑(由 SDPJDetector 完成);不做响应内容的业务语义解析

18. 按报告ID查询检测结果数据
    - 输入:检测报告ID
    - 输出:该报告下全部结果数据明细
    - 触发场景:ReportManager 生成可视化报告时加载明细数据(对应 1.spec.md 功能 1.5);用户查看/编辑报告时(对应 1.spec.md 功能 1.6)

19. 删除检测结果数据条目
    - 输入:结果数据ID
    - 输出:删除结果
    - 触发场景:用户精修报告或清理异常记录(对应 1.spec.md 功能 1.6)

# 接口契约
20. 通过 ResultDBInterface 对外暴露上述能力,DataProcessor 是唯一调用方(符合 4.模型依赖关系图.puml 中 DataProcessor → ResultDB 边)

不需要的:[维护结果数据版本/修订历史,执行统计分析或聚合等业务逻辑,做权限判定,做跨库外键完整性校验(用户ID/数据集ID),做调用方业务逻辑(分页/排序/筛选聚合),做报告可视化渲染,维护大模型业务元数据]
依赖模块:无
应实现接口:ResultDBInterface
被依赖模块:DataProcessor
技术细节:(暂定)


实体:
检测样本数据[检测样本ID,风险具体子类,漏洞概念验证数据]
安全风险检测数据集[数据集ID,数据集名称,安全风险类型]
检测任务[任务ID,任务状态,开始时间,结束时间]
检测任务组[任务组ID]
检测报告[结果数据ID,风险具体子类,被测大模型输出内容,合规判断结果]
用户[用户ID,账号,密码]
被测大模型[模型ID]

关系:
检测样本数据→安全风险检测数据集:包含(n-1)
检测任务→检测任务组:包含(n-1)
检测任务组→安全风险检测数据集:使用(1-n)
检测任务→检测报告:生成(1-1)
用户→检测任务组:创建(1-n)
检测结果数据→检测报告:包含(n-1)
被测大模型→检测任务组:使用(1-1)

在E-R图的基础上，进一步设计符合数据库规范性3NF标准的关系模型，得:
检测任务组(任务组ID[主键], 用户ID[外键_用户信息数据库模块.用户(用户ID)], 模型ID[外键_被测大模型(模型ID)])
检测任务(任务ID[主键], 任务组ID[外键_检测任务组(任务组ID)], 数据集ID[外键_检测样本数据库模块.安全风险检测数据集(数据集ID)], 任务状态, 开始时间, 结束时间)
被测大模型(模型ID[主键])
检测报告(检测报告ID[主键], 任务ID[外键_检测任务(任务ID)])
检测结果数据(结果数据ID[主键], 检测报告ID[外键_检测报告(检测报告ID)], 风险具体子类, 被测大模型输出内容, 合规判断结果)
在此基础上进行数据表设计，得到对应的五张数据表，即可实现该模块基本功能。
