SampleDB / 检测样本数据库模块
具体职责:

# 数据集级能力
1. 创建检测数据集
   - 输入:数据集名称、安全风险类型
   - 输出:新创建数据集的数据集 ID
   - 触发场景:用户上传私有数据集(对应 1.spec.md 功能 3.1.1.2)
   - 错误情形:数据集名称在系统内重复时拒绝写入
   - 不负责的边界:不解析数据集文件内容、不做格式校验(由 DataProcessor 与 UtilsLib 完成)

2. 删除检测数据集
   - 输入:数据集 ID
   - 输出:删除结果
   - 后置条件:级联删除该数据集下的所有样本(满足外键完整性)
   - 触发场景:用户删除私有数据集
   - 不负责的边界:不做调用方权限判定(由 DACManager 在调用前完成)

3. 查询数据集列表
   - 输入:无
   - 输出:全部数据集元信息列表(数据集 ID、名称、安全风险类型、created_at)
   - 触发场景:用户选择检测数据集时(对应 1.spec.md 功能 1.3)
   - 不负责的边界:不做分页/排序业务逻辑(由调用方按需处理)

4. 按 ID 查询单个数据集
   - 输入:数据集 ID
   - 输出:数据集元信息(数据集 ID、名称、安全风险类型、created_at);不存在时通过明确错误反馈
   - 触发场景:DataProcessor 在执行检测前定位特定数据集

5. 按安全风险类型筛选数据集
   - 输入:安全风险类型(如「越狱攻击」「提示词注入」「安全基准」)
   - 输出:匹配的数据集列表(每条含数据集 ID、名称、安全风险类型、created_at)
   - 触发场景:DataProcessor 加载特定类型数据集(如静态算法步骤 1 加载 D_jailbreak)

# 样本级能力
6. 添加检测样本到指定数据集
   - 输入:风险具体子类 ST、漏洞概念验证数据 PoC、所属数据集 ID
   - 输出:新创建样本的样本 ID
   - 错误情形:所属数据集 ID 不存在时拒绝写入
   - 触发场景:DataProcessor 在用户上传私有数据集时,逐条写入样本
   - 不负责的边界:不做多模态/多编码处理(由 UtilsLib 完成);不解析 PoC 语义

7. 删除检测样本
   - 输入:样本 ID
   - 输出:删除结果
   - 触发场景:用户删除单条私有样本

8. 按数据集 ID 查询所有样本
   - 输入:数据集 ID
   - 输出:该数据集下全部样本列表(每条含样本 ID、风险具体子类 ST、漏洞概念验证数据 PoC、created_at)
   - 触发场景:SDPJDetector 通过 DataProcessor 加载某个 D_i 的全部样本(静态/动态算法步骤 1 与步骤 3 均使用)

9. 按 ID 查询单条样本
   - 输入:样本 ID
   - 输出:单条样本详情(ST、PoC、所属数据集 ID、created_at)

检测样本数据库模块E-R图序列化数据： 
实体1：检测样本数据 
  属性：样本ID、风险具体子类、漏洞概念验证数据 
实体2：安全风险检测数据集 
  属性：数据集ID、数据集名称、安全风险类型 
关系：包含 
  关联实体及基数：检测样本数据（n） ↔ 安全风险检测数据集（1） 
  语义：1个数据集包含n个检测样本数据；1个检测样本数据属于1个数据集
在E-R图的基础上，进一步设计符合数据库规范性3NF标准的关系模型，得:
安全风险检测数据集(数据集ID[主键], 数据集名称, 安全风险类型)
检测样本数据(样本ID[主键], 风险具体子类, 漏洞概念验证数据, 数据集ID)
在此基础上进行数据表设计，得到对应的两张数据表，即可实现该模块基本功能。
