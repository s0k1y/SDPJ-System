UserCenter / 用户管理中心模块
具体职责:

# 账号生命周期
1. 注册新用户
   - 输入:账号、明文密码
   - 输出:新用户的用户 ID
   - 后置条件:用户记录已持久化,存储密码为不可逆摘要形式
   - 触发场景:用户注册(对应 1.spec.md 功能 3.1.1.1)
   - 调用下层:UtilsLib 的「哈希摘要」对密码做单向哈希后,调用 UserDB 的「创建用户」
   - 不负责的边界:不做账号格式与密码强度校验(由 AccountManager 完成);不做加解密密钥的持有与生命周期管理

2. 注销用户
   - 输入:用户 ID
   - 输出:注销结果
   - 后置条件:该用户及其拥有的资源、相关 ACL 条目全部清理(由下层级联)
   - 触发场景:用户注销账号(对应 1.spec.md 功能 3.1.1.2)
   - 调用下层:UserDB 的「删除用户」

3. 修改用户密码
   - 输入:用户 ID、新明文密码
   - 输出:修改结果
   - 后置条件:存储密码已更新为新密码的不可逆摘要
   - 触发场景:用户修改密码(对应 1.spec.md 功能 3.1.1.2)
   - 调用下层:UtilsLib 的「哈希摘要」与 UserDB 的「更新用户密码」

4. 按账号查询用户信息
   - 输入:账号
   - 输出:用户信息(用户 ID、账号,不向上透出存储密码)
   - 触发场景:账号切换(对应 1.spec.md 功能 3.1.1.2)
   - 调用下层:UserDB 的「按账号查询用户」

5. 按用户 ID 查询用户信息
   - 输入:用户 ID
   - 输出:用户信息(用户 ID、账号,不向上透出存储密码)
   - 触发场景:AccountManager 读取当前用户信息、账号切换(对应 1.spec.md 功能 3.1.1.2);ReportManager 查询报告归属用户(对应 1.spec.md 功能 1.5 / 1.6)
   - 调用下层:UserDB 的「按 ID 查询用户」

# 凭据校验
6. 登录凭据校验
   - 输入:账号、明文密码
   - 输出:校验结果(通过时附带用户 ID)
   - 触发场景:用户登录(对应 1.spec.md 功能 3.1.1.1)
   - 调用下层:UserDB 的「按账号查询用户」读取存储凭据,UtilsLib 的「哈希摘要」做比对
   - 不负责的边界:不建立、不维护登录会话与在线态

# 资源登记与查询
7. 登记受控资源
   - 输入:资源类型、拥有者用户 ID
   - 输出:新创建资源的资源 ID
   - 触发场景:用户创建受控资源(如私有检测配置文件、私有数据集等)时为其建立 ACL 承载点(对应 1.spec.md 功能 3.1.1.2)
   - 调用下层:UserDB 的「登记资源」
   - 不负责的边界:不维护资源实体的具体内容(仅维护元信息与 ACL)

8. 移除受控资源
   - 输入:资源 ID
   - 输出:移除结果
   - 后置条件:该资源相关全部 ACL 条目、若有对应私有检测配置内容均由下层级联清理
   - 触发场景:用户删除受控资源(对应 1.spec.md 功能 3.1.1.2)
   - 调用下层:UserDB 的「删除资源」

9. 按拥有者查询资源清单
   - 输入:用户 ID
   - 输出:该用户拥有的全部资源列表(每条含资源 ID、资源类型)
   - 触发场景:AccountManager 列出当前用户的受控资产(对应 1.spec.md 功能 3.1.1.2)
   - 调用下层:UserDB 的「按拥有者查询资源列表」

10. 按 ID 查询资源
    - 输入:资源 ID
    - 输出:资源信息(资源类型、拥有者用户 ID)
    - 触发场景:DACManager 在权限判定前定位资源归属(对应 1.spec.md 功能 3.1.2)
    - 调用下层:UserDB 的「按 ID 查询资源」

# ACL 授权管理
11. 授予他人访问权
    - 输入:资源 ID、被授权用户 ID
    - 输出:新创建的访问控制项 ID
    - 触发场景:资源拥有者对其他用户授予访问权(对应 1.spec.md 功能 3.1.2)
    - 调用下层:UserDB 的「添加访问控制项」
    - 不负责的边界:不判定调用者是否具备「授权他人」资格(由 DACManager 在调用前完成)

12. 移除已授予的访问权
    - 输入:访问控制项 ID
    - 输出:移除结果
    - 触发场景:资源拥有者收回对他人的授权(对应 1.spec.md 功能 3.1.2)
    - 调用下层:UserDB 的「删除访问控制项」

13. 查询某资源的授权清单
    - 输入:资源 ID
    - 输出:该资源下全部访问控制项(每条含访问控制项 ID、被授权用户 ID)
    - 触发场景:DACManager 展示资源授权状况(对应 1.spec.md 功能 3.1.2)
    - 调用下层:UserDB 的「按资源ID查询访问控制项列表」

14. 判定用户对资源是否具备访问权
    - 输入:资源 ID、用户 ID
    - 输出:是否具备访问权的布尔判定
    - 触发场景:DACManager 校验某用户对某资源的访问权(对应 1.spec.md 功能 3.1.2)
    - 调用下层:UserDB 的「判定访问控制项是否存在」

# 私有检测配置内容管理
15. 写入用户私有检测配置内容
    - 输入:配置ID(等同于其对应资源的资源ID)、配置内容JSON
    - 输出:写入结果
    - 触发场景:PrivateConfigManager 在创建私有检测配置后持久化其内容(对应 1.spec.md 功能 3.1.1.2)
    - 调用下层:UserDB 的「写入私有检测配置内容」

16. 读取用户私有检测配置内容
    - 输入:配置ID
    - 输出:配置内容JSON
    - 触发场景:PrivateConfigManager 在用户查看或系统加载私有检测配置时(对应 1.spec.md 功能 1.1 / 1.2 / 1.3 / 3.1.1.2)
    - 调用下层:UserDB 的「按 ID 读取私有检测配置内容」

17. 更新用户私有检测配置内容
    - 输入:配置ID、新配置内容JSON
    - 输出:更新结果
    - 触发场景:PrivateConfigManager 在用户编辑已有私有配置后写入新内容(对应 1.spec.md 功能 3.1.1.2)
    - 调用下层:UserDB 的「更新私有检测配置内容」

18. 清除用户私有检测配置内容
    - 输入:配置ID
    - 输出:清除结果
    - 触发场景:作为显式清除路径,通常由资源删除级联完成(对应 1.spec.md 功能 3.1.1.2)
    - 调用下层:UserDB 的「删除私有检测配置内容」

# 接口契约
19. 通过 UserCenterInterface 对外暴露上述能力,PrivateConfigManager / ReportManager / AccountManager / DACManager 为调用方(符合 4.模型依赖关系图.puml 中四条 → UserCenter 的有向边)

不需要的:[不维护用户登录会话与在线态,不做账号格式/密码强度校验(由 AccountManager 完成),不持有或管理加解密密钥的生命周期(密钥由外部安全组件提供),不做权限授予资格的业务判定(由 DACManager 完成),不维护受控资源的具体内容(仅维护元信息与 ACL),不向上透出存储密码等敏感字段,不做调用方业务逻辑(分页/排序/筛选)]
依赖模块:UserDB,UtilsLib,调用接口:UserDBInterface,UtilsInterface
应实现接口:UserCenterInterface
被依赖模块:PrivateConfigManager,ReportManager,AccountManager,DACManager
技术细节:(暂定)
