UserCenter / 用户管理中心模块
具体职责:

# 账号生命周期
1. 注册新用户
   - 输入:账号、明文密码
   - 输出:新用户的用户 ID
   - 后置条件:用户记录已持久化
   - 触发场景:用户注册(对应 1.spec.md 功能 3.1.1.1)
   - 调用下层:UserDB 的「创建用户」
   - 不负责的边界:不做账号格式与密码强度校验(由 AccountManager 完成)

2. 注销用户
   - 输入:用户 ID
   - 输出:注销结果
   - 后置条件:该用户及其拥有的资源、相关 ACL 条目全部清理(由下层级联)
   - 触发场景:用户注销账号(对应 1.spec.md 功能 3.1.1.2)
   - 调用下层:UserDB 的「删除用户」

3. 修改用户密码
   - 输入:用户 ID、新明文密码
   - 输出:修改结果
   - 触发场景:用户修改密码(对应 1.spec.md 功能 3.1.1.2)
   - 调用下层:UserDB 的「更新用户密码」

4. 修改用户名
   - 输入:用户 ID、新用户名
   - 输出:修改结果
   - 触发场景:用户修改个人资料中的用户名(对应 1.spec.md 功能 3.1.1.2)
   - 调用下层:UserDB 的「更新用户名」

5. 获取所有用户列表
   - 输入:无
   - 输出:全部用户信息列表(用户 ID、账号,不向上透出存储密码)
   - 触发场景:管理员浏览系统用户、StateScheduler 获取全体用户列表用于管理操作
   - 调用下层:UserDB 的「查询所有用户」

6. 按账号查询用户信息
   - 输入:账号
   - 输出:用户信息(用户 ID、账号,不向上透出存储密码)
   - 触发场景:账号切换(对应 1.spec.md 功能 3.1.1.2)
   - 调用下层:UserDB 的「按账号查询用户」

7. 按用户 ID 查询用户信息
   - 输入:用户 ID
   - 输出:用户信息(用户 ID、账号,不向上透出存储密码)
   - 触发场景:AccountManager 读取当前用户信息、账号切换(对应 1.spec.md 功能 3.1.1.2);ReportManager 查询报告归属用户(对应 1.spec.md 功能 1.5 / 1.6)
   - 调用下层:UserDB 的「按 ID 查询用户」

# 凭据校验
8. 登录凭据校验
   - 输入:账号、明文密码
   - 输出:三元组 (是否校验通过, 用户ID(通过时)或None(失败时), 描述信息)
   - 返回类型:tuple[bool, Optional[int], str]
   - 触发场景:用户登录(对应 1.spec.md 功能 3.1.1.1)
   - 调用下层:UserDB 的「按账号查询用户」读取存储凭据,直接比对密码
   - 不负责的边界:不建立、不维护登录会话与在线态
   - 实现说明:规范原描述为"返回用户 ID 或校验失败标识",实际实现返回三元组以同时携带校验成功/失败布尔值、用户 ID 和可读描述信息,便于调用方统一处理各种校验结果场景(成功、用户不存在、密码错误等)。

# 资源登记与查询
9. 登记受控资源
   - 输入:资源类型、拥有者用户 ID
   - 输出:新创建资源的资源 ID
   - 触发场景:用户创建受控资源(如私有检测配置文件、私有数据集等)时为其建立 ACL 承载点(对应 1.spec.md 功能 3.1.1.2)
   - 调用下层:UserDB 的「登记资源」
   - 不负责的边界:不维护资源实体的具体内容(仅维护元信息与 ACL)

10. 移除受控资源
   - 输入:资源 ID
   - 输出:移除结果
   - 后置条件:该资源相关全部 ACL 条目、若有对应私有检测配置内容均由下层级联清理
   - 触发场景:用户删除受控资源(对应 1.spec.md 功能 3.1.1.2)
   - 调用下层:UserDB 的「删除资源」

11. 按拥有者查询资源清单
   - 输入:用户 ID
   - 输出:该用户拥有的全部资源列表(每条含资源 ID、资源类型)
   - 触发场景:AccountManager 列出当前用户的受控资产(对应 1.spec.md 功能 3.1.1.2)
   - 调用下层:UserDB 的「按拥有者查询资源列表」

12. 按 ID 查询资源
    - 输入:资源 ID
    - 输出:资源信息(资源类型、拥有者用户 ID)
    - 触发场景:DACManager 在权限判定前定位资源归属(对应 1.spec.md 功能 3.1.2)
    - 调用下层:UserDB 的「按 ID 查询资源」

# ACL 授权管理
13. 授予他人访问权
    - 输入:资源 ID、被授权用户 ID
    - 输出:新创建的访问控制项 ID
    - 触发场景:资源拥有者对其他用户授予访问权(对应 1.spec.md 功能 3.1.2)
    - 调用下层:UserDB 的「添加访问控制项」
    - 不负责的边界:不判定调用者是否具备「授权他人」资格(由 DACManager 在调用前完成)

14. 移除已授予的访问权
    - 输入:访问控制项 ID
    - 输出:移除结果
    - 触发场景:资源拥有者收回对他人的授权(对应 1.spec.md 功能 3.1.2)
    - 调用下层:UserDB 的「删除访问控制项」

15. 查询某资源的授权清单
    - 输入:资源 ID
    - 输出:该资源下全部访问控制项(每条含访问控制项 ID、被授权用户 ID)
    - 触发场景:DACManager 展示资源授权状况(对应 1.spec.md 功能 3.1.2)
    - 调用下层:UserDB 的「按资源ID查询访问控制项列表」

16. 判定用户对资源是否具备访问权
    - 输入:资源 ID、用户 ID
    - 输出:是否具备访问权的布尔判定
    - 触发场景:DACManager 校验某用户对某资源的访问权(对应 1.spec.md 功能 3.1.2)
    - 调用下层:UserDB 的「判定访问控制项是否存在」

17. 查询与某用户共享的资源清单
    - 输入:用户 ID
    - 输出:该用户被授权访问的全部资源列表(含资源 ID、资源类型、授权来源)
    - 触发场景:DACManager 展示其他用户共享给当前用户的资源(对应 1.spec.md 功能 3.1.2)
    - 调用下层:UserDB 的「按被授权用户查询共享资源」

18. 按 ID 查询访问控制项
    - 输入:访问控制项 ID
    - 输出:访问控制项详情(含资源 ID、被授权用户 ID)
    - 触发场景:DACManager 在撤销授权前定位具体的访问控制项(对应 1.spec.md 功能 3.1.2)
    - 调用下层:UserDB 的「按 ID 查询访问控制项」

19. 获取用户可访问的资源 ID 集合
    - 输入:用户 ID
    - 输出:该用户可访问的全部资源 ID 集合
    - 触发场景:DACManager 批量校验用户对多项资源的访问权,通常用于过滤列表展示(对应 1.spec.md 功能 3.1.2)
    - 调用下层:UserDB 的「查询用户可访问的资源 ID 列表」

# 私有检测配置内容管理
20. 写入用户私有检测配置内容
    - 输入:配置ID(等同于其对应资源的资源ID)、配置内容JSON
    - 输出:写入结果
    - 触发场景:PrivateConfigManager 在创建私有检测配置后持久化其内容(对应 1.spec.md 功能 3.1.1.2)
    - 调用下层:UserDB 的「写入私有检测配置内容」

21. 读取用户私有检测配置内容
    - 输入:配置ID
    - 输出:配置内容JSON
    - 触发场景:PrivateConfigManager 在用户查看或系统加载私有检测配置时(对应 1.spec.md 功能 1.1 / 1.2 / 1.3 / 3.1.1.2)
    - 调用下层:UserDB 的「按 ID 读取私有检测配置内容」

22. 更新用户私有检测配置内容
    - 输入:配置ID、新配置内容JSON
    - 输出:更新结果
    - 触发场景:PrivateConfigManager 在用户编辑已有私有配置后写入新内容(对应 1.spec.md 功能 3.1.1.2)
    - 调用下层:UserDB 的「更新私有检测配置内容」

23. 清除用户私有检测配置内容
    - 输入:配置ID
    - 输出:清除结果
    - 触发场景:作为显式清除路径,通常由资源删除级联完成(对应 1.spec.md 功能 3.1.1.2)
    - 调用下层:UserDB 的「删除私有检测配置内容」

24. 批量读取用户私有检测配置内容
    - 输入:配置ID列表
    - 输出:配置ID到配置内容JSON的映射字典
    - 触发场景:PrivateConfigManager 需要批量加载多项私有检测配置时,减少逐条读取的数据库往返开销
    - 调用下层:UserDB 的「批量读取私有检测配置内容」

# 接口契约
25. 通过 UserCenterInterface 对外暴露上述能力,PrivateConfigManager / ReportManager / AccountManager / DACManager 为调用方(符合 4.模型依赖关系图.puml 中四条 → UserCenter 的有向边)

不需要的:[不维护用户登录会话与在线态,不做账号格式/密码强度校验(由 AccountManager 完成),不持有或管理加解密密钥的生命周期(密钥由外部安全组件提供),不做权限授予资格的业务判定(由 DACManager 完成),不维护受控资源的具体内容(仅维护元信息与 ACL),不向上透出存储密码等敏感字段,不做调用方业务逻辑(分页/排序/筛选)]
依赖模块:UserDB,调用接口:UserDBInterface
应实现接口:UserCenterInterface
被依赖模块:PrivateConfigManager,ReportManager,AccountManager,DACManager
技术细节:(暂定)
