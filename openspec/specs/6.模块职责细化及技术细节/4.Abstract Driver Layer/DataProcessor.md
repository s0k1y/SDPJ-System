DataProcessor / 检测数据处理模块
具体职责:

# 检测样本与数据集的查询与维护
1. 查询检测数据集清单
   - 输入:无
   - 输出:全部数据集元信息列表(数据集 ID、名称、安全风险类型)
   - 触发场景:用户选择要使用的检测数据集(对应 1.spec.md 功能 1.3)
   - 调用下层:SampleDB 的「查询数据集列表」
   - 不负责的边界:不做调用方分页/排序/按业务属性过滤

2. 按安全风险类型加载数据集样本
   - 输入:安全风险类型(如「越狱攻击」「提示词注入」「安全基准」等)
   - 输出:对应数据集及其全部样本(每条含样本 ID、风险具体子类 ST、漏洞概念验证数据 PoC)
   - 触发场景:SDPJ 静态/动态算法在步骤 1/步骤 3 加载 D_i 的全部样本(对应 1.spec.md 功能 1.1 / 1.2)
   - 调用下层:SampleDB 的「按安全风险类型筛选数据集」与「按数据集 ID 查询所有样本」
   - 不负责的边界:不做样本/PoC 的业务语义审查(由 SDPJDetector 完成);不做数据缓存;不做合规判断(由 SDPJDetector 完成)

3. 导入用户私有数据集
   - 输入:数据集元信息(名称、安全风险类型)、样本条目清单(已完成格式预校验)
   - 输出:新创建数据集 ID 及各新增样本 ID
   - 触发场景:用户上传私有数据集(对应 1.spec.md 功能 3.1.1.2)
   - 调用下层:SampleDB 的「创建检测数据集」与「添加检测样本到指定数据集」
   - 不负责的边界:不做文件格式校验(由 UtilsLib 在上游完成);不做调用方权限判定(由 DACManager 完成)

4. 移除私有数据集或其单条样本
   - 输入:数据集 ID 或样本 ID
   - 输出:删除结果
   - 后置条件:删除数据集时由下层级联清理其下全部样本
   - 触发场景:用户移除私有数据集或单条样本(对应 1.spec.md 功能 3.1.1.2)
   - 调用下层:SampleDB 的「删除检测数据集」/「删除检测样本」

# 检测结果的持久化
5. 开设检测任务组
   - 输入:用户 ID、目标被测大模型 ID
   - 输出:新创建任务组的任务组 ID
   - 后置条件:若被测大模型未登记则先行入表
   - 触发场景:用户启动一次检测时建立承载本次检测所有产出的顶层条目(对应 1.spec.md 功能 1.1 / 1.2)
   - 调用下层:ResultDB 的「登记被测大模型」与「创建检测任务组」

6. 创建检测子任务
   - 输入:任务组 ID、数据集 ID、初始任务状态、开始时间
   - 输出:新创建任务的任务 ID
   - 触发场景:静态/动态算法在每个待测数据集 D_i 上各启动一次检测子任务(对应 1.spec.md 功能 1.1 / 1.2)
   - 调用下层:ResultDB 的「创建检测任务」

7. 更新检测子任务状态
   - 输入:任务 ID、任务状态、结束时间(可选)
   - 输出:更新结果
   - 触发场景:任务由「进行中」转入「完成」或「异常中断」时
   - 调用下层:ResultDB 的「更新检测任务状态」

8. 创建检测报告条目
   - 输入:所属任务 ID
   - 输出:新创建报告的检测报告 ID
   - 触发场景:一条检测子任务完成后为其生成对应报告条目(对应 1.spec.md 功能 1.1 / 1.2 收尾)
   - 调用下层:ResultDB 的「创建检测报告」

9. 追加单条检测结果数据
   - 输入:检测报告 ID、风险具体子类、被测大模型输出内容、合规判断结果
   - 输出:新创建结果条目的结果数据 ID
   - 触发场景:每完成对一条样本的检测后将模型响应与合规结论落盘(对应 1.spec.md 功能 1.1 / 1.2)
   - 调用下层:ResultDB 的「追加检测结果数据条目」
   - 不负责的边界:不做合规判断逻辑(由 SDPJDetector 完成)

# 检测报告数据的汇总与产出
10. 汇总任务组下全部检测明细
    - 输入:任务组 ID
    - 输出:任务组 → 任务 → 报告 → 结果明细的结构化数据
    - 触发场景:ReportManager 生成可视化检测结果报告(对应 1.spec.md 功能 1.5)
    - 调用下层:ResultDB 的「按任务组ID查询任务列表」「按任务ID查询检测报告」「按报告ID查询检测结果数据」
    - 不负责的边界:不做可视化渲染(由 ReportManager 完成)

11. 查询检测任务组/报告清单
    - 输入:过滤条件(用户 ID / 模型 ID 等,可选)
    - 输出:任务组与报告元信息列表
    - 触发场景:用户浏览历史检测(对应 1.spec.md 功能 1.6)
    - 调用下层:ResultDB 的「查询检测任务组列表」与「查询检测报告列表」

12. 删除检测报告及其关联结果
    - 输入:任务组 ID / 任务 ID / 检测报告 ID / 结果数据 ID(按删除粒度任选其一)
    - 输出:删除结果
    - 后置条件:按删除粒度由下层保证级联清理
    - 触发场景:用户删除检测报告或精修报告内容(对应 1.spec.md 功能 1.6)
    - 调用下层:ResultDB 对应粒度的删除能力(「删除检测任务组」/「删除检测任务」/「删除检测报告」/「删除检测结果数据条目」)

13. 导出检测报告文件
    - 输入:已汇总的报告结构化数据、目标文件格式
    - 输出:可供用户下载的文件内容
    - 触发场景:用户下载检测报告(对应 1.spec.md 功能 1.6)
    - 调用下层:UtilsLib 的「结构化数据序列化」与「文件写入」

# 多模态与多编码的样本构造及响应处理
14. 构造多模态检测样本
    - 输入:文本形式的漏洞概念验证数据 PoC、目标模态(图像/音频/视频)
    - 输出:目标模态形式的样本数据
    - 触发场景:静态/动态算法构造多模态注入样本(对应 1.spec.md 功能 1.1 / 1.2 中的多模态注入检测场景)
    - 调用下层:UtilsLib 的「文本到多模态数据的生成」

15. 解析大模型多模态响应
    - 输入:大模型返回的多模态响应内容
    - 输出:从中提取的文本内容
    - 触发场景:对大模型多模态回复的后处理(对应 1.spec.md 功能 1.1 / 1.2)
    - 调用下层:UtilsLib 的「多模态数据到文本的反向解析」

16. 构造多编码检测样本
    - 输入:文本形式的 PoC、目标编码形式(如 Base64、URL 编码、Unicode 转义、十六进制等)
    - 输出:目标编码下的样本文本
    - 触发场景:静态/动态算法构造多编码注入样本(对应 1.spec.md 功能 1.1 / 1.2 中的多编码注入检测场景)
    - 调用下层:UtilsLib 的「文本编码转换」

17. 还原编码响应内容
    - 输入:大模型响应中的编码文本、已知编码形式
    - 输出:还原后的原始文本
    - 触发场景:对大模型响应中的编码内容做还原(对应 1.spec.md 功能 1.1 / 1.2)
    - 调用下层:UtilsLib 的「编码文本还原」

# 文件导入及结构化数据序列化
18. 读取本地数据集/配置文件
    - 输入:文件路径或文件流、读取模式(文本/二进制)
    - 输出:文件内容
    - 触发场景:加载数据集文件(对应 1.spec.md 功能 1.1 / 1.2);加载用户私有检测配置文件(功能 3.1.1.2)
    - 调用下层:UtilsLib 的「文件读取」

19. 预校验上传文件格式
    - 输入:文件内容、期望格式(如 CSV / JSON / 特定 schema)
    - 输出:格式合规性判定结果及关键错误信息
    - 触发场景:用户上传私有数据集/私有检测配置文件时的格式预校验(对应 1.spec.md 功能 3.1.1.2)
    - 调用下层:UtilsLib 的「文件格式校验」
    - 不负责的边界:不做内容业务语义审查

20. 结构化数据的序列化与反序列化
    - 输入:结构化对象或序列化字符串、目标/源格式(如 JSON / YAML)
    - 输出:转换结果
    - 触发场景:跨层结构化数据(私有检测配置、报告汇总数据等)的文件落盘与载入
    - 调用下层:UtilsLib 的「结构化数据序列化」/「数据反序列化」

# 接口契约
21. 通过 DataProcessorInterface 对外暴露上述能力,SDPJDetector / PrivateConfigManager / ReportManager 为调用方(符合 4.模型依赖关系图.puml 中三条 → DataProcessor 的有向边)

不需要的:[不做合规判断逻辑(由 SDPJDetector 完成),不做报告可视化渲染(由 ReportManager 完成),不做调用方权限判定(由 DACManager 完成),不维护大模型业务元数据(由 LLMRegistry 完成),不维护数据缓存,不做样本/PoC 的业务语义解析,不做调用方业务逻辑(分页/排序/筛选聚合),不做跨库外键完整性校验(由调用方在调用前保证)]

依赖模块:SampleDB,ResultDB,UtilsLib,调用接口:SampleDBInterface,ResultDBInterface,UtilsInterface
应实现接口:DataProcessorInterface
被依赖模块:SDPJDetector,PrivateConfigManager,ReportManager
