## 新增需求

### 需求: Linux Shell启动脚本
系统 SHALL 提供Linux/macOS Shell启动脚本（start.sh），实现与Windows PowerShell脚本等价的启动功能。

#### 场景: 成功启动系统
- **当** 用户执行 `./start.sh` 命令时
- **则** 系统初始化SSL证书、数据库，清理端口，启动后端和前端服务

#### 场景: 脚本执行权限
- **当** 用户首次运行start.sh脚本时
- **则** 系统提示用户添加执行权限 `chmod +x start.sh`

#### 场景: Python环境检测
- **当** 用户执行start.sh但Python未安装时
- **则** 系统显示错误提示"Python 3.11+未安装"

### 需求: SSL证书初始化
系统 SHALL 在启动时自动初始化SSL证书，确保HTTPS服务可用。

#### 场景: 证书自动生成
- **当** 启动脚本执行时证书文件不存在
- **则** 系统通过SecureCommManager自动生成自签名证书

#### 场景: 证书已存在
- **当** 启动脚本执行时证书文件已存在
- **则** 系统跳过证书生成步骤，使用现有证书

### 需求: 数据库初始化
系统 SHALL 在首次启动时自动初始化数据库。

#### 场景: 数据库不存在
- **当** 启动脚本执行时数据库文件不存在
- **则** 系统运行seed_data脚本创建数据库和初始数据

#### 场景: 数据库已存在
- **当** 启动脚本执行时数据库文件已存在
- **则** 系统跳过数据库初始化步骤

### 需求: 端口清理
系统 SHALL 在启动前清理占用目标端口的进程。

#### 场景: 端口被占用
- **当** 启动脚本执行时端口8000或5173被占用
- **则** 系统自动终止占用端口的进程

#### 场景: 端口清理失败
- **当** 端口清理操作失败时
- **则** 系统显示错误提示并尝试使用备用端口

### 需求: 服务启动
系统 SHALL 按顺序启动后端和前端服务。

#### 场景: 后端服务启动
- **当** 端口清理完成后
- **则** 系统在后台启动FastAPI后端服务（端口8000）

#### 场景: 前端服务启动
- **当** 后端服务启动后
- **则** 系统在后台启动Vue前端开发服务器（端口5173）

#### 场景: 启动完成提示
- **当** 所有服务启动完成后
- **则** 系统显示服务访问地址和API文档链接
