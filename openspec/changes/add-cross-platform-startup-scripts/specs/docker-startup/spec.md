## 新增需求

### 需求: Docker一键部署
系统 SHALL 提供Docker容器化部署方案，支持通过docker-compose命令一键启动前后端服务。

#### 场景: 成功启动Docker服务
- **当** 用户执行 `docker-compose up -d` 命令时
- **则** 系统启动后端FastAPI服务（端口8000）和前端Nginx服务（端口80）

#### 场景: Docker环境未安装
- **当** 用户执行docker-compose命令但Docker未安装时
- **则** 系统显示错误提示"Docker未安装，请先安装Docker Desktop"

#### 场景: 端口冲突处理
- **当** Docker服务启动时端口8000或80已被占用
- **则** 系统显示端口冲突错误并提供解决方案提示

### 需求: Docker服务健康检查
系统 SHALL 为Docker服务提供健康检查机制，确保服务正常运行。

#### 场景: 后端服务健康检查
- **当** 后端容器启动后
- **则** 系统每30秒检查 `/health` 端点，确保服务响应正常

#### 场景: 前端服务健康检查
- **当** 前端容器启动后
- **则** 系统每30秒检查Nginx服务状态，确保服务响应正常

### 需求: Docker日志管理
系统 SHALL 提供Docker服务日志查看和管理功能。

#### 场景: 查看服务日志
- **当** 用户执行 `docker-compose logs -f` 命令时
- **则** 系统显示前后端服务的实时日志输出

#### 场景: 日志持久化
- **当** Docker服务运行时
- **则** 系统将日志文件持久化到宿主机的 `./logs` 目录
