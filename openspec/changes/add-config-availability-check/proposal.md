# Proposal: add-config-availability-check

## 变更名称
检测配置文件可用性验证

## 动机
用户上传的大模型适配器配置文件可能存在以下问题，但当前系统无法在上传后、检测前发现：
1. **request_format 与 API 不匹配**：如 DeepSeek 使用 OpenAI 格式但配置了 `request_format: "anthropic"`，导致所有 API 调用 404 但被静默吞掉
2. **API Key 无效或过期**：密钥错误导致认证失败
3. **base_url 错误**：端点地址不可达
4. **模型名称错误**：模型 ID 不存在

当前用户只能在启动检测后才发现配置不可用，且错误被静默处理，排查困难。需要在配置管理阶段提供可用性验证能力。

## 目标
1. 在私有配置页面增加"测试连接"按钮，用户可在保存配置后验证其可用性
2. 后端新增配置可用性验证 API，实际调用目标大模型并返回验证结果
3. 验证结果应包含：连接状态、模型响应摘要、错误信息（如有）、响应延迟

## 范围
- 后端：StateScheduler 新增 `verify_config_availability` 方法，detection router 新增 `/api/detection/config/verify` 端点
- 前端：PrivateConfig.vue 配置列表操作栏增加"测试"按钮，展示验证结果
- 不涉及：修改现有检测流程、修改适配器加载逻辑

## 验收标准
1. 用户在私有配置页面点击"测试"按钮后，系统向目标大模型发送一条简短测试消息
2. 验证成功时显示：连接成功、模型名称、响应延迟(ms)
3. 验证失败时显示：错误类型（认证失败/端点不可达/格式不匹配/超时等）、错误详情
4. 验证过程有加载状态指示，超时时间 30 秒
5. 不影响系统当前状态（验证不触发 FSM 状态转移）
